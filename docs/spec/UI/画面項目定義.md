# RideIntercom 画面項目定義

## 目的

本書は現行実装の UI 項目を、画面単位かつ親子関係が分かる形で整理した正本である。  
各項目について、表示名だけでなく背景、意味、意図、表示条件、データ仕様、表示仕様を記載し、UI 上の値が何を表すかを読み取れる状態にする。

## 記載方針

| 項目 | 方針 |
|---|---|
| 対象画面 | `ContentView.swift` に実装される `Call`、`Diagnostics`、`Settings` |
| 表先頭列 | UI項目を定義する表は、先頭列を `項目名（日本語）` に統一する |
| 親子関係 | 親セクションと子項目を分離し、所属関係が追える表構成にする |
| ラベル | UI に表示される文字列、またはラベルを持たない場合は役割名で記載する |
| データ仕様 | 値の型、値域、参照元、表示用整形方法を記載する |
| 表示仕様 | 見え方、レイアウト、色、操作方法、補助表示の意味を記載する |
| 状態 | 接続状態、音声状態、診断状態など、ユーザーが画面から読み取るべき状態変化を記載する |
| 異常系 | エラー表示や無効化条件など、通常系から外れる時の見え方を記載する |

## 共通定義

| 項目 | 内容 |
|---|---|
| ナビゲーション | `TabView` + 各タブ内 `NavigationStack` |
| ローカルメンバー | `selectedGroup.members.first` を自分として扱う |
| リモートメンバー | `selectedGroup.members.dropFirst()` を相手一覧として扱う |
| メーター表示 | `VoiceMeterView` は値文字列を直接出さず、レベルとピークを視覚表現する |
| URL 受理 | `rideintercom://join?token=...` を `onOpenURL` で受理し、成功時は Call タブへ戻す |

| コンテキスト削除 | `group-row` | `group-row-context-delete` | `Delete` | ポインタ/長押し操作でも削除できるようにするため | スワイプできない操作系向けの代替破壊操作 | 対象 `group.id` | 各行常時 | context menu 項目 | 行削除 | 削除導線を複線化するが意味は同じ |

### 子項目: ステータスヘッダー

| 項目名（日本語） | 親項目 | 項目ID | ラベル | 背景 | 意味・意図 | データ仕様 | 発現条件 | 表示仕様 | 状態 | 異常系の考え方 |

| 接続状態ラベル | `call-connection-summary` | `call-presence-label` | 接続状態名 | 操作直後の状態変化を読み取りやすくするため | 現在の通話プレゼンス状態を示す主文言 | `String` | 常時 | 1 行目テキスト | 接続有無、待機、再接続中、接続済みだが音声未開始 (`Connected / Audio Idle`) など | 状態不明時も空欄ではなく文言化する |

| ローカル入力ヘッダー | `call-status-header` | `local-microphone-header` | `Input` | 自分の送信状態を即座に確認できるようにするため | ローカル入力の生死、レベル、ミュートを表す領域 | `GroupMember`, `Bool` | `localMember != nil` | ローカル入力状態カード | Live / Muted とメーターが連動 | ローカルメンバー未確定時は表示しない |

| ローカル入力メーター | `local-microphone-header` | `local-microphone-meter` | なし | 自分の入力レベルを視覚確認するため | マイクが反応しているか、発話が取れているかを確認する | `voiceLevel`, `voicePeakLevel`, `isMuted` | 常時 | 値文字列なしのメーター | 入力レベルとピーク | ミュート中は実質 0 表示として扱う |
| ローカルミュートボタン | `local-microphone-header` | `local-microphone-mute-button` | `Mute` / `Unmute` | 片手操作で送話を止められるようにするため | 自分の送信停止/再開を切り替える操作 | `Bool -> String` | 常時 | アイコン単体ボタン | 押下でミュート状態が反転 | 音声処理の都合で再開失敗しても状態表示を優先する |
| マスター出力グループ | `call-status-header` | `master-output-group` | `Output` | インターコム音声だけを相対的に調整できるようにするため | 受信音声全体に対する最終出力ゲインを扱う | `masterOutputVolume`, `isOutputMuted` | 常時 | 出力ラベル + スライダー + ミュートボタン | 音量値とミュート状態 | OS 音量ではなくアプリ内最終出力を対象とする |
| マスター出力スライダー | `master-output-group` | `master-output-slider` | `Output Volume` | 他アプリ音量を上げず相対調整したいため | インターコムの最終出力レベルを調整する | `Float`、範囲 `0...2` | 常時 | `1.0` が通常値、`1.0` 超過は boost | 通常域とブースト域を兼ねる | 過大入力時も最終サンプルはソフトクリップされる前提 |
| マスター出力ミュートボタン | `master-output-group` | `master-output-mute-button` | `Mute Output` / `Unmute Output` | 受信のみ一時停止したい場面に備えるため | 再生側最終ゲインを 0 にする操作 | `Bool -> String` | 常時 | ミュート時は赤系 | 出力ミュート状態 | 入力や接続は止めず、出力だけを止める |
| 接続状態アイコン | `call-connection-summary` | `call-connection-icon` | なし | 文言を読まなくても接続段階を判別できるようにするため | 接続確立、接続途中、未接続、異常を SF Symbol で表す | `selectedGroupConnectionState`, `isAudioReady`, `canDisconnectCall` | 常時 | 状態文言の左に 1 つ表示 | `wifi` 系 / `exclamationmark.triangle.fill` を使い分ける | 通信方式や control-only は別の補助表示で扱い、接続成否アイコンへ混在させない |
| ダッキング状態アイコン | `master-output-group` | `call-ducking-status-icon` | なし | 他音声ダック設定が有効か、かつ現在実際に発動中かを通話画面で即判別できるようにするため | 他音声ダックの設定状態と実効状態を SF Symbol で表す | `supportsAdvancedMixingOptions`, `isDuckOthersEnabled`, `isOtherAudioDuckingActive` | `supportsAdvancedMixingOptions == true` の時のみ | `Output` 見出しの補助位置に 1 つ表示 | OFF は非表示、設定 ON かつ非発動中は `waveform` を補助色、設定 ON かつ発動中は `waveform` を `info` または `success` 系で表示 | 非対応 platform では項目自体を出さない。発動条件を満たさない時は「未発動」であり異常扱いしない |

#### 接続状態アイコン詳細

| 項目 | 内容 |
|---|---|
| 対象実装 | `CallView.connectionIconName` / `connectionStatusColor` |
| 参照定義 | `docs/design system/アイコン定義.md` の `wifi` / `exclamationmark.triangle.fill` |
| 未接続 | `selectedGroupConnectionState == .idle` の時は `wifi.slash` + `neutral` |
| 接続中 | `selectedGroupConnectionState == .localConnecting || .internetConnecting` の時は `wifi.exclamationmark` + `warning` |
| 接続済み | `selectedGroupConnectionState == .localConnected || .internetConnected` の時は `wifi` + `success` |
| 異常 | `selectedGroupConnectionState == .reconnectingOffline` の時は `exclamationmark.triangle.fill` + `danger` |
| 補助通信方式表示 | `routeLabel` の左に小さい補助アイコンを置く。`Local` 系は `dot.radiowaves.left.and.right`、`Internet` 系は `network`。色は常に `textSecondary` |

#### ダッキング状態アイコン詳細

| 項目 | 内容 |
|---|---|
| 対象実装 | `CallView` の `master-output-group` 内。`duckingStatusIcon` |
| 参照定義 | `docs/design system/アイコン定義.md` の `Audio Processing / waveform` |

## 表示解釈上の方針

| 項目 | 方針 |
|---|---|
| Input メーター | ローカルは送信前入力、参加者カードは受信後デコード済み入力を示す |
| Output メーター | Settings と Diagnostics の出力値は、実際に再生へ回した最終出力を示す |
| Participants 状態 | 接続イベントだけでなく、受信、jitter、再生の進行を含めて読む |
| Diagnostics 行 | 1 行ごとに「どの層の状態か」を分け、通話不良の切り分けに使う |
