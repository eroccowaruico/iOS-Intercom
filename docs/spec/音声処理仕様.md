# RideIntercom 音声処理仕様

## 目的

本仕様は、現行実装における音声処理の実行経路、状態遷移、主要パラメータ、異常時動作を定義する。  
対象は Call 実行時の双方向音声と Settings の Audio Check 経路とする。

## 適用範囲

| 区分 | 対象 | 備考 |
|---|---|---|
| 送信系 | マイク入力〜VAD〜エンコード〜送信 | Local/Internet 共通フロー |
| 受信系 | 受信〜ジッタバッファ〜ミックス〜出力 | リモート話者ごとの音量適用を含む |
| 診断系 | Audio Check 録音/再生、Diagnostics 表示値 | 実機チェック用 |
| 非対象 | サーバーサイド処理 | 本アプリ範囲外 |

## 音声処理アーキテクチャ

### 実行フロー（通常通話）

| ステップ | 入力 | 処理 | 出力 | 主な責務 |
|---|---|---|---|---|
| Mic Capture | AVAudioEngine input tap | RMS算出、サンプル抽出、16kHzへリサンプル | level, samples | マイク信号取得 |
| VAD | level | ノイズフロア追従+閾値判定 | idle/attack/talking/release | 発話区間判定 |
| Packetize | frameID, samples, VAD状態 | pre-roll付与、keepalive生成 | OutboundAudioPacket | 送信単位生成 |
| Encode | samples | codec別エンコード | EncodedVoicePacket | 帯域削減 |
| Transport | packet | local/internetへ送信 | 暗号化済み転送データ | 通信層送出 |
| Receive | 受信データ | 復号、デコード、検証 | ReceivedAudioPacket | 受信正規化 |
| Jitter | packet+timestamp | playout遅延制御、有効期限破棄 | Ready frames | 途切れ抑制 |
| Output | ready frames | ミックス、出力ゲイン適用、再生 | スピーカー出力 | 可聴音声再生 |

### 主要コンポーネント

| コンポーネント | 役割 |
|---|---|
| IntercomViewModel | 通話状態管理、各サービス接続、UI向け状態公開 |
| AudioSessionManager | AudioSession適用、I/Oポート設定、有効化/無効化 |
| SystemAudioInputMonitor | 入力tap、RMS計測、Sound Isolation切替 |
| AudioTransmissionController | VAD制御、pre-roll、keepalive生成 |
| AudioPacketSequencer | sequence採番、codecエンコード、envelope化 |
| RemoteAudioPipelineService/JitterBuffer | 受信パケット蓄積、再生タイミング決定 |
| AudioFramePlayer | フレームミックスと出力レンダラーへの供給 |

## セッション設定

| 用途 | category | mode | options |
|---|---|---|---|
| Intercom通話 | playAndRecord | voiceChat | mixWithOthers, allowBluetooth, allowBluetoothA2DP |
| Audio Check | playAndRecord | default | allowBluetooth, allowBluetoothA2DP |

## 送信仕様

### 入力前処理

| 項目 | 値/仕様 |
|---|---|
| input tap bufferSize | 128 |
| サンプルレート正規化 | 16,000 Hzへリサンプル |
| 入力レベル | RMS（0.0〜1.0へクランプ） |
| Sound Isolation | 対応OSで切替可能（未対応時はN/A） |

### VAD仕様

| 項目 | 値 |
|---|---|
| 状態 | idle / attack / talking / release |
| defaultThreshold | 0.003 |
| minThreshold | 0.00025 |
| maxThreshold | 0.12 |
| attackFrames | 1 |
| releaseFrames | 40 |
| noiseFloor初期値 | minThreshold |

補足: start/release判定は固定閾値とnoiseFloor追従値の大きい方を使い、静音環境と騒音環境の双方で誤検出を抑制する。

### パケット生成仕様

| 項目 | 値 |
|---|---|
| preRoll保持数 | 20 frames |
| keepalive送信間隔 | 50 frames |
| frameID | 1から連番 |
| mute時動作 | 音声送信停止、ローカル入力メーター0化 |

## コーデック仕様

| codec | 利用可否 | 備考 |
|---|---|---|
| PCM 16-bit | 利用可 | 既定選択 |
| HE-AAC v2 VBR | 利用可 | 品質プリセットでbitrate切替 |
| Opus | 利用可 | AVFoundation backend利用時に選択可能 |

### HE-AAC v2 詳細

| 項目 | 値 |
|---|---|
| sampleRate | 16,000 Hz |
| channel | 2ch（mono入力をL/R複製） |
| frameSize | 2,048 samples |
| bitrate Low | 16,000 bps |
| bitrate Medium | 24,000 bps |
| bitrate High | 40,000 bps |
| strategy | Variable Bit Rate |

## 受信仕様

| 項目 | 値/仕様 |
|---|---|
| ジッタバッファ再生遅延 | 0.015 sec |
| パケット保持寿命 | 2.0 sec |
| 重複パケット | streamID+sequenceNumberで破棄 |
| フレーム順序 | streamID→sequenceNumberで整列 |
| 期限切れパケット | 破棄カウント加算 |

## 出力仕様

| 項目 | 値/仕様 |
|---|---|
| 出力サンプルレート | 16,000 Hz |
| 出力チャンネル数 | 1ch |
| マスター出力音量 | 0.0〜1.0 |
| 参加者別出力音量 | 0.0〜1.0 |
| 出力ミュート | ゲイン0適用 |
| サンプルクランプ | -1.0〜1.0 |

## Audio Check 仕様

| ステップ | 処理 | 時間 |
|---|---|---|
| recording | マイク入力を収集 | 5秒 |
| playing | 録音データをcodec経由で再生 | 5秒 |
| completed | 完了状態へ遷移 | 即時 |

| 項目 | 仕様 |
|---|---|
| 開始条件 | recording/playing中でないこと |
| 記録サンプル0件時 | failed（No microphone samples captured） |
| 再生codec | preferredTransmitCodecがPCM/HE-AAC/利用可能なOpus時は同一、利用不可なOpus指定時はPCMへフォールバック |
| 占有動作 | 通話未開始時のみAudio Checkがセッションを所有し、完了時に解放 |

## 異常時仕様

| 事象 | 表示/挙動 |
|---|---|
| マイク権限未決定 | permission request pendingメッセージを表示し開始失敗 |
| マイク権限拒否 | Privacy設定を促すメッセージを表示し開始失敗 |
| 入出力ポート変更失敗 | Audio input/output device change failed |
| codec利用不可 | 送信時はPCMへフォールバック、失敗時はkeepalive化 |

## 状態管理仕様

| 項目 | 仕様 |
|---|---|
| mute自動停止 | mute継続2秒でマイクキャプチャ停止 |
| mute解除時 | 可能ならマイク再開 |
| remote talker失効 | 0.6秒無音受信でtalking解除 |
| diagnostics値 | 通話中とAudio Check中で入力/出力メーター参照元を切替 |
