# RideIntercom 画面・状態遷移

## 目的

本ドキュメントは、現行実装における画面遷移と主要状態遷移を可視化する。  
実装者・テスターが同じ遷移モデルで確認できるよう、Mermaid図で定義する。

## 画面遷移（Tab + Call内部）

```mermaid
flowchart LR
	A[App起動] --> B[TabView]
	B --> C[Callタブ]
	B --> D[Diagnosticsタブ]
	B --> E[Settingsタブ]

	C --> F{selectedGroup == nil}
	F -- Yes --> G[Groups画面]
	F -- No --> H[Call画面]

	G -- Create Trail Group --> H
	G -- Group選択 --> H
	H -- Groupsボタン --> G

	H -- Connect --> I[接続処理]
	H -- Disconnect --> J[切断処理]
	I --> H
	J --> H

	H -- Invite URL共有 --> H
	D --> D
	E --> E
```

## Call 接続状態遷移

対象: CallConnectionState

```mermaid
stateDiagram-v2
	[*] --> idle

	idle --> localConnecting: Connect押下 + local connect開始
	localConnecting --> localConnected: local connected/authenticated

	localConnecting --> internetConnecting: local linkFailed(internetAvailable=true)
	localConnected --> internetConnecting: local linkFailed(internetAvailable=true)

	localConnecting --> reconnectingOffline: local linkFailed(internetAvailable=false)
	localConnected --> reconnectingOffline: local linkFailed(internetAvailable=false)

	internetConnecting --> internetConnected: internet connected/authenticated
	internetConnecting --> reconnectingOffline: internet linkFailed

	internetConnected --> localConnected: local candidate + probe成功 + handover完了
	reconnectingOffline --> localConnected: local再接続成功
	reconnectingOffline --> internetConnected: internet再接続成功

	idle --> idle: selectedGroup未選択
	localConnected --> idle: Disconnect / showGroupSelection
	internetConnected --> idle: Disconnect / showGroupSelection
	internetConnecting --> idle: Disconnect / showGroupSelection
	reconnectingOffline --> idle: Disconnect / showGroupSelection
```

## 経路制御状態遷移

対象: RouteCoordinator.phase

```mermaid
stateDiagram-v2
	[*] --> idle

	idle --> localConnected: connectLocal

	localConnected --> internetConnecting: localLinkDidFail(internetAvailable=true)
	localConnected --> reconnectingOffline: localLinkDidFail(internetAvailable=false)

	internetConnecting --> internetConnected: internetDidConnect
	internetConnected --> localCandidate: localCandidateDetected
	internetConnected --> localProbing: beginLocalProbe

	localCandidate --> localProbing: beginLocalProbe
	localCandidate --> internetConnected: probeWindow timeout(7.5s)
	localProbing --> internetConnected: probeWindow timeout(7.5s)

	localProbing --> handoverToLocal: shouldPreferLocal == true
	handoverToLocal --> localConnected: dualSendWindow timeout(1.0s)

	reconnectingOffline --> idle: disconnect
	localConnected --> idle: disconnect
	internetConnecting --> idle: disconnect
	internetConnected --> idle: disconnect
	localCandidate --> idle: disconnect
	localProbing --> idle: disconnect
	handoverToLocal --> idle: disconnect
```

## Audio Check 状態遷移

対象: AudioCheckPhase

```mermaid
stateDiagram-v2
	[*] --> idle

	idle --> recording: Record 5s and Play
	recording --> failed: setup失敗 or samples empty
	recording --> playing: 5秒録音完了

	playing --> completed: 5秒再生完了

	failed --> recording: Record 5s and Play
	completed --> recording: Record 5s and Play

	recording --> recording: 実行中はボタンdisabled
	playing --> playing: 実行中はボタンdisabled
```

## Local Network 状態遷移

対象: LocalNetworkStatus

```mermaid
stateDiagram-v2
	[*] --> idle

	idle --> advertisingBrowsing: localTransport.connect
	advertisingBrowsing --> invited: peer発見 + groupHash一致
	advertisingBrowsing --> invitationReceived: 招待受信
	advertisingBrowsing --> rejected: groupHash不一致
	advertisingBrowsing --> unavailable: advertise/browse開始失敗

	invited --> connected: MC接続成功
	invitationReceived --> connected: MC接続成功

	connected --> idle: local disconnect
	rejected --> idle: 再試行 or 再接続
	unavailable --> idle: 再試行 or disconnect
```

## VAD 状態遷移

対象: VoiceActivityState

```mermaid
stateDiagram-v2
	[*] --> idle

	idle --> attack: 入力レベル >= startThreshold かつ attackFrames未満
	attack --> talking: attackFrames到達
	attack --> idle: 閾値未満

	talking --> release: release判定区間
	release --> talking: releaseThresholdより上
	release --> idle: silenceFramesがreleaseFrames到達

	talking --> talking: 発話継続
	idle --> idle: 無音継続
```
