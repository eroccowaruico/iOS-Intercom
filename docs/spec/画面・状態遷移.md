# RideIntercom 画面・状態遷移

## 目的

本書は現行実装の画面遷移と主要状態遷移を示す。  
対象は `ContentView.swift` と `IntercomCore.swift` に存在する遷移のみであり、未実装構想は含めない。

## 画面遷移

```mermaid
flowchart LR
    A[App起動] --> B[TabView]
    B --> C[Call]
    B --> D[Diagnostics]
    B --> E[Settings]

    C --> F{selectedGroup == nil}
    F -- yes --> G[Groups画面]
    F -- no --> H[Call画面]

    G -- Create Talk Group --> H
    G -- グループ行タップ --> H
    H -- Groupsボタン --> G

    H -- Invite URL受理成功 --> H
    D --> D
    E --> E
```

## Call 画面内の業務遷移

```mermaid
flowchart TD
    A[Groups画面] -->|selectGroup| B[グループ選択済み]
    B -->|startLocalStandby| C[Local待受]
    B -->|Connect| D[通話接続開始]
    D -->|Disconnect| B
    D -->|showGroupSelection| A
    D -->|Share Invite| E[共有シート]
    E -->|TapGesture| F[招待枠1件予約]
    F --> D
```

補足:

| 項目 | 現行仕様 |
|---|---|
| グループ選択直後 | 音声は開始せず、`startLocalStandby()` で Local 探索のみ開始 |
| Connect 押下 | 音声パイプライン開始後に接続状態をアクティブ化 |
| Invite | 共有シートを開くのみ。QR 画面や専用招待画面はない |

## CallConnectionState

対象: `CallConnectionState`

```mermaid
stateDiagram-v2
    [*] --> idle

    idle --> localConnecting: Connect押下かつ connectedPeerIDs が空
    idle --> localConnected: Connect押下時点で connectedPeerIDs が存在

    localConnecting --> localConnected: local authenticated / connected
    localConnecting --> internetConnecting: localLinkDidFail(true)
    localConnecting --> reconnectingOffline: localLinkDidFail(false)

    localConnected --> internetConnecting: localLinkDidFail(true)
    localConnected --> reconnectingOffline: localLinkDidFail(false)

    internetConnecting --> internetConnected: internetDidConnect
    internetConnecting --> reconnectingOffline: linkFailed(false)

    internetConnected --> localConnected: local probe成功 + handover完了
    reconnectingOffline --> localConnected: local再接続成功
    reconnectingOffline --> internetConnected: internet再接続成功

    localConnecting --> idle: disconnect / showGroupSelection
    localConnected --> idle: disconnect / showGroupSelection
    internetConnecting --> idle: disconnect / showGroupSelection
    internetConnected --> idle: disconnect / showGroupSelection
    reconnectingOffline --> idle: disconnect / showGroupSelection
```

## RouteCoordinator.phase

対象: `RouteCoordinator.Phase`

```mermaid
stateDiagram-v2
    [*] --> idle

    idle --> localConnected: connectLocal

    localConnected --> internetConnecting: localLinkDidFail(true)
    localConnected --> reconnectingOffline: localLinkDidFail(false)

    internetConnecting --> internetConnected: internetDidConnect
    internetConnected --> localCandidate: localCandidateDetected
    internetConnected --> localProbing: beginLocalProbe

    localCandidate --> localProbing: beginLocalProbe
    localCandidate --> internetConnected: deadline超過
    localProbing --> internetConnected: deadline超過

    localProbing --> handoverToLocal: shouldPreferLocal == true
    handoverToLocal --> localConnected: dualSendWindow超過

    reconnectingOffline --> idle: disconnect
    localConnected --> idle: disconnect
    internetConnecting --> idle: disconnect
    internetConnected --> idle: disconnect
    localCandidate --> idle: disconnect
    localProbing --> idle: disconnect
    handoverToLocal --> idle: disconnect
```

## AudioCheckPhase

対象: `AudioCheckPhase`

```mermaid
stateDiagram-v2
    [*] --> idle

    idle --> recording: startAudioCheck
    recording --> failed: setup失敗 / samples empty
    recording --> playing: 5秒録音完了

```

## LocalNetworkStatus

対象: `LocalNetworkStatus`

```mermaid
stateDiagram-v2
    [*] --> idle

    idle --> advertisingBrowsing: localTransport.connect
    advertisingBrowsing --> invited: groupHash一致の peer を発見
    advertisingBrowsing --> invitationReceived: 招待受信
    advertisingBrowsing --> connected: MC接続確立
    advertisingBrowsing --> rejected: group mismatch
    advertisingBrowsing --> unavailable: advertise/browse開始失敗

    invited --> connected: MC接続確立
    invitationReceived --> connected: MC接続確立

    connected --> idle: disconnect
    rejected --> idle: 再接続または画面離脱
    unavailable --> idle: disconnect
```

## VoiceActivityState

対象: `VoiceActivityState`

```mermaid
stateDiagram-v2
    [*] --> idle

    idle --> attack: 入力 >= startThreshold かつ attackFrames 未満
    attack --> talking: attackFrames 到達
    attack --> idle: 閾値未満へ戻る

    talking --> release: release 判定へ移行
    release --> talking: releaseThreshold 超過
    release --> idle: silenceFrames が releaseFrames 到達
```

## 補助状態

| 状態 | 遷移概要 |
|---|---|
| `selectedGroup` | `nil` なら Groups 画面、非 `nil` なら Call 画面 |
| `audioCheckOwnsAudioPipeline` | 通話未開始時に Audio Check がセッションを所有し、完了/失敗時に解放 |
| `isMicrophoneCaptureSuspendedByMute` | ミュート継続 2 秒後に `true` となり、解除時に再開を試行 |
