# RideIntercom 設定値一覧

## 目的

本書は現行実装で扱う設定値と状態値を、変更可否と反映タイミングで整理する。

## ユーザー操作で変更できる値

| 設定名 | 識別子 | 型 | 既定値 | 範囲/候補 | 反映タイミング | UI |
|---|---|---|---|---|---|---|
| 入力ポート | `selectedInputPort` | `AudioPortInfo` | `Auto` | 利用可能入力ポート | セッション起動中は即時、それ以外は次回起動時 | Settings > Audio I/O > Input |
| 出力ポート | `selectedOutputPort` | `AudioPortInfo` | `Auto` | iOS: `Auto`, `Speaker` / macOS: `Auto`, 利用可能出力デバイス | セッション起動中は即時、それ以外は次回起動時 | Settings > Audio I/O > Output |
| Sound Isolation | `isSoundIsolationEnabled` | `Bool` | `true` | `true/false` | 即時。voice processing 状態変更時は入力ノードと tap を再構成して反映 | Settings > Audio I/O > Sound Isolation |
| 他音声ダック | `isDuckOthersEnabled` | `Bool` | `false` 想定 | `true/false` | iOS は `enableAdvancedDucking: true` を常用し、`duckingLevel` で強度を制御する。ON かつ可聴な受信音声がある時は `.default`、それ以外は `.min`。受信停止後約 1 秒で `.min` へ戻す。出力ミュート、マスター出力 0、対象 peer 出力 0 の時は `.min` を維持。マイクミュート状態と独立して動作する。macOS は UI 非表示で固定 | Settings > Audio I/O > Duck Other Audio |
| VAD 閾値 | `voiceActivityDetectionThreshold` | `Float` | `0.003` | `0.00025 ... 0.12` | 即時 | Settings > VAD Threshold |
| Audio Check 実行 | `startAudioCheck()` | Action | なし | 実行のみ | ボタン押下時 | Settings > Audio Check |
| 設定リセット | `resetAllSettings()` | Action | なし | 実行のみ | ボタン押下時 | Settings > Reset Settings |
| マスター出力音量 | `masterOutputVolume` | `Float` | `1.0` | `0.0 ... 2.0` | 即時 | Call > Output slider |
| マスター出力ミュート | `isOutputMuted` | `Bool` | `false` | `true/false` | 即時 | Call > Output mute |
| 参加者別出力音量 | `remoteOutputVolumes[peerID]` | `Float` | 未設定時 `1.0` | `0.0 ... 1.0` | 即時 | Call > Participant Output slider |
| ローカルマイクミュート | `isMuted` | `Bool` | `false` | `true/false` | 即時。入力エンジンは停止せず `isVoiceProcessingInputMuted` で VP 入力を抑止する | Call > Input mute |

## UI には存在するが実質固定の値

| 項目 | 識別子 | UI 上の見え方 | 現行実装 |
|---|---|---|---|
| 送信 codec | `preferredTransmitCodec` | Settings に segmented picker を表示 | setter が常に `.pcm16` を設定し、picker も disabled |
| HE-AAC 品質 | `heAACv2Quality` | HE-AAC 選択時のみ表示の設計 | setter が常に `.medium` を設定。通常フローでは HE-AAC に入らない |

## 永続化される値

| 項目 | 保存先 | 備考 |
|---|---|---|
| `RideIntercom.localMemberID` | `UserDefaults` | 初回生成後に保持 |
| `RideIntercom.localDisplayName` | `UserDefaults` | 初回は `You` |
| グループ一覧 | `UserDefaults` | `RideIntercom.groups`。`accessSecret` は保存しない |
| グループ secret | Keychain | `RideIntercom.GroupCredential` サービス |

## 永続化されない値

| 項目 | 理由 |
|---|---|
| 音声 I/O 選択 | プロセス内状態のみ |
| 他音声ダック | プロセス内状態のみ |
| Spoken Audio 割り込み | プロセス内状態のみ |
| VAD 閾値 | プロセス内状態のみ |
| Sound Isolation | プロセス内状態のみ |
| 出力音量/ミュート | プロセス内状態のみ |
| 参加者別出力音量 | プロセス内状態のみ |
| Audio Check 状態 | 実行時状態のみ |

## 設定リセット対象

| 項目 | リセット後の値 |
|---|---|
| 入力ポート | `Auto` |
| 出力ポート | `Auto` |
| Sound Isolation | `ON` |
| 他音声ダック | `false` |
| VAD 閾値 | `0.003` |
| 送信 codec | `PCM 16-bit` |
| HE-AAC 品質 | `Medium` |
| マスター出力音量 | `100%` |
| マスター出力ミュート | `false` |
| 参加者別出力音量 | 未設定扱い（各参加者 `100%`） |

`resetAllSettings()` はグループ一覧、参加者構成、保存済みメンバー、グループ secret を変更しない。

## 内部固定値

| 項目 | 識別子 | 値 |
|---|---|---|
| VAD 最小閾値 | `VoiceActivityDetector.minThreshold` | `0.00025` |
| VAD 最大閾値 | `VoiceActivityDetector.maxThreshold` | `0.12` |
| VAD 既定閾値 | `VoiceActivityDetector.defaultThreshold` | `0.003` |
| VAD attack | `attackFrames` | `1` |
| VAD release | `releaseFrames` | `40` |
| pre-roll | `preRollLimit` | `20 frames` |
| keepalive 間隔 | `keepaliveIntervalFrames` | `50 frames` |
| mute 自動停止 | 廃止 | ミュート時はエンジンを停止せず `isVoiceProcessingInputMuted` で入力を抑止する。ducking の動的制御に入力エンジンの維持が必要なため |
| remote talker timeout | `remoteTalkerTimeout` | `0.6 sec` |
| jitter delay | `JitterBuffer` の playout delay | `0.015 sec` |
| jitter lifetime | `JitterBuffer` packet lifetime | `2.0 sec` |
| 入力 tap | `installTap` bufferSize | `128` |
| 入出力 sample rate | 入力リサンプル先 / 出力 renderer | `16_000 Hz` |
| 出力 channel | `SystemAudioOutputRenderer` | `1 ch` |
| Audio Check 録音時間 | `startAudioCheck(recordDuration:)` 既定 | `5 sec` |
| Audio Check 再生時間 | `startAudioCheck(playbackDuration:)` 既定 | `5 sec` |
| Local probe window | `RouteCoordinator.probeWindow` | `7.5 sec` |
| dual send window | `RouteCoordinator.dualSendWindow` | `1.0 sec` |

## 通信系で参照する設定

| 設定 | 通信への影響 |
|---|---|
| 入力ポート | 送信元音声の物理ソースを変える |
| VAD 閾値 | voice / keepalive の生成頻度に影響 |
| ローカルマイクミュート | voice 送信停止と RideIntercom application data による mute 同期 |
| Sound Isolation | 入力信号特性に影響 |
| 送信 codec 表示値 | envelope metadata に影響するが、実送信は PCM 固定 |
| 出力系設定 | 通信ではなく再生ゲインにのみ影響 |
