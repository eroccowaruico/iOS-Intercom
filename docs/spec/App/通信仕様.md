# RideIntercom App 通信仕様

## 目的

本書は、作り直す RideIntercom App が RTC package をどう使うかだけを定義する。

接続、経路選択、Multipeer、WebRTC、packet audio、handover、暗号化の詳細は `docs/spec/packages/RTC.md` を正とし、本書では App 本体に残る責務だけを扱う。

## 責務境界

| 領域 | App 本体の責務 | 詳細仕様 |
|---|---|---|
| グループ | group ID、表示名、group secret の保存と選択 | 本書 |
| ローカルメンバー | local member ID、display name の生成と保存 | 本書 |
| 招待 | join URL の生成、共有、受理 | 本書 |
| RTC 接続 | RTC へ credential、peer、route 設定を渡す | `docs/spec/packages/RTC.md` |
| 音声 media | media 開始/停止を App の音声 pipeline と同期する | `docs/spec/App/音声処理仕様.md`、`docs/spec/packages/RTC.md` |
| App data | RideIntercom 固有 namespace と payload schema を管理する | 本書 |
| UI 表示 | 接続状態、参加者、エラー要約を表示する | `docs/spec/App/UI` |

## 基本方針

| 項目 | 方針 |
|---|---|
| RTC 詳細 | App は route、packet、handshake、WebRTC native 型を直接扱わない |
| 接続と media | 接続成立と音声開始を分ける |
| 認証 | group secret 由来の credential を RTC へ渡し、認証結果は RTC event として受け取る |
| 設定 | RTC へ渡す route / codec / credential 設定は本書に重複定義せず、`docs/spec/App/setting parameters` に集約する。通信 route の opt-out は Settings の `enabledRTCTransportRoutes` を RTC package configuration へ渡す |
| runtime status | RTC の package-owned runtime status を App data より先に decode し、通信相手の package 状態として保持する |
| 機密情報 | group secret と invite token はログへ出さない |
| ログ | RTC event / route metrics は `docs/requirements/log.md` と `docs/spec/App/ログ仕様.md` に従って構造化ログへ変換する |

## App が保持する通信状態

| 状態 | 内容 |
|---|---|
| local member | この端末の member ID と表示名 |
| group list | 参加済みグループの ID と表示名 |
| group secret | 各 group の secret。Keychain に保存する |
| selected group | 画面で選択中の group |
| active group | RTC 接続中または接続準備中の group |
| members | RTC event と App data から得た参加者表示状態 |
| connection summary | RTC の接続状態を UI 用に要約したもの |
| remote runtime statuses | `RTCRuntimeStatusTransport` で受信した peer ごとの package runtime status |
| enabled RTC routes | Settings で選択した RTC route の有効集合。既定は `.multipeer` と `.webRTC` の両方を有効にし、App は空集合を既定 route へ修復して RTC package へ渡さない |

## 設定値

RTC へ渡す route、credential、peer、codec UI 設定、Application Data namespace の設定は `docs/spec/App/setting parameters/App/設定値一覧.md` を正とする。

RTC package の型名や内部設定の詳細は `docs/spec/packages/RTC.md` を正とする。

## 接続操作

| 操作 | App の動作 |
|---|---|
| group 選択 | active group がなければ選択 group を画面へ反映し、credential と local peer を RTC へ渡して接続を開始する。既存の active group は勝手に切り替えない |
| connect | 切断済みの選択 group を再接続する |
| route opt-out | Settings で Local Network / Internet toggle を変更した場合、App は更新後の `enabledRTCTransportRoutes` を `CallSession.setEnabledRoutes(_:)` へ渡し、active RTC connection があれば停止する。最後の有効 route は OFF にできず、旧設定などで空集合を読んだ場合は既定 route へ修復する |
| media 開始 | RTC が media 開始可能になったら App の音声 pipeline を開始する |
| codec 変更 | App の `preferredTransmitCodec` を RTC / Codec adapter へ反映し、route 未対応または codec unavailable の場合は fallback 結果を Diagnostics へ表示する |
| disconnect | RTC 接続と App の音声 pipeline を停止する |
| group 切替 | active group がある場合は明示操作で切断してから別 group へ接続する |
| 画面遷移 | 画面を戻るだけでは接続を切らない |

## RTC error の扱い

| RTC event | App の扱い |
|---|---|
| route scoped error | `connectionFailed(.webRTC, ...)` など route 単体の失敗は通話全体の `linkFailed` に変換しない。Diagnostics / runtime status の入力として扱い、RTC package の fallback / restore に任せる |
| `noEnabledRoute` | App 設定修復後も発生した場合は通話全体の失敗として扱う |
| `stateChanged(.failed)` | 通話全体の失敗として扱い、`connectionState = .reconnectingOffline` へ反映する |

## 招待 URL

App はグループ参加用に `rideintercom://join?token=...` を扱う。

| 項目 | 仕様 |
|---|---|
| URL scheme | `rideintercom://join` |
| query | `token` |
| token 形式 | base64url JSON |
| 生成元 | 共有対象 group |
| 受理先 | App の deep link 入口 |
| 失敗時 | group へ参加せず、受理失敗として表示する |

### token 内容

| フィールド | 内容 |
|---|---|
| `version` | token 形式の version |
| `groupID` | 参加先 group ID |
| `groupName` | 参加先 group 表示名 |
| `groupSecret` | RTC credential 生成に使う secret |
| `inviterMemberID` | 招待元 member ID |
| `issuedAt` | 発行時刻 |
| `expiresAt` | 失効時刻 |
| `signature` | 改ざん検知用署名 |

### 受理フロー

| 手順 | 動作 |
|---|---|
| decode | URL から token を復元する |
| validate | version、signature、expiresAt を検証する |
| store | group 情報を保存し、group secret を Keychain に保存する |
| select | 受理した group を選択状態にする |
| report | 成功または失敗を UI に表示する |

## App data

RTC の `ApplicationDataMessage` は任意 payload の配送だけを担当する。RideIntercom 固有の namespace と payload schema は App が管理する。

| namespace | delivery | 所有者 | 用途 |
|---|---|---|---|
| `rideintercom.rtc.runtimeStatus` | RTC package が決める | RTC package | 接続、route、media、codec、local control、他 package runtime report を同期する |
| `rideintercom.keepalive` | unreliable | App | peer の lightweight metadata を同期する |
| `rideintercom.peerMuteState` | reliable | App | ローカルマイクミュート状態を相手画面へ同期する |

RTC package-owned namespace は App 固有 schema より先に `RTCRuntimeStatusTransport.decode(_:)` で判定する。decode できた場合は `remoteRuntimeStatuses[peerID]` へ保持し、App data として再解釈しない。

### `rideintercom.rtc.runtimeStatus`

| フィールド | App の扱い |
|---|---|
| `connectionState` / `isMediaStarted` | remote peer の control / media 状態として保持する |
| `routeSnapshot` / `routes` | active route、media route、route capability、media ownership を package 情報として保持する |
| `audioCodecConfiguration` / `routes.selectedAudioCodec` | remote peer の codec 設定と実選択 codec として保持する |
| `localMute` / `outputMute` / `remoteOutputVolumes` | remote peer が公開した local control 状態として保持する |
| `packageReports` | SessionManager、Codec、AudioMixer、Effectors など他 package の report として保持する |

### `rideintercom.keepalive`

| フィールド | 内容 |
|---|---|
| `activeCodec` | peer が最後に選択した codec。受信側の `GroupMember.activeCodec` に反映する |

### `rideintercom.peerMuteState`

| フィールド | 内容 |
|---|---|
| `isMuted` | 送話ミュート状態 |

未対応 namespace は無視する。未認証 peer からの App data 受理可否は RTC package の責務とする。

## Package runtime report 送信

App は RTC 以外の package runtime report を集め、`CallSession.updateRuntimePackageReports(_:)` で RTC へ渡す。RTC は payload を解釈せず、`RTCRuntimeStatus.packageReports` として package-owned runtime status に同梱する。

| package | report kind | payload |
|---|---|---|
| AudioMixer | `snapshot` | `AudioMixerSnapshot` |
| Codec | `runtimeReport` | `CodecRuntimeReport` |
| VADGate | `runtimeSnapshot` | `VADGateRuntimeSnapshot` |
| SoundIsolation | `transmitRuntimeSnapshot` / `receiveMasterRuntimeSnapshot` | `VoiceIsolationRuntimeSnapshot` |
| DynamicsProcessor | `transmitRuntimeSnapshot` | `DynamicsProcessorRuntimeSnapshot` |
| PeakLimiter | `transmitRuntimeSnapshot` / `receiveMasterRuntimeSnapshot` | `PeakLimiterRuntimeSnapshot` |
| SessionManager | `audioSessionConfigurationRequest` | `AudioSessionConfiguration` |
| SessionManager | `audioSessionConfigurationReport` | `AudioSessionConfigurationReport` |
| SessionManager | `inputStreamOperationReport` / `outputStreamOperationReport` / `voiceProcessingOperationReport` | `AudioStreamOperationReport` |

| 送信契機 | App の動作 |
|---|---|
| 接続開始 / standby / 認証 / 切断 | report を強制更新し RTC へ渡す |
| media start / media stop | 音声 pipeline の開始停止結果を report に反映して渡す |
| 設定変更 | codec、bitrate、VAD、mute、ducking、output mute、master volume、peer volume、SoundIsolation を変更した時点で渡す |
| SessionManager report 受信 | session / stream / voice processing report を保持して即時渡す |
| 通話 tick | 最短 0.5 秒間隔で最新 report を渡す |

RTC package は接続開始、route変更、media開始/停止、local control変更、package report更新、定期周期で `rideintercom.rtc.runtimeStatus` を送る。App はこの status を、通信相手の設定や操作状態の正本として扱う。

## 表示する通信状態

| 表示 | 意味 |
|---|---|
| 未接続 | RTC 接続がない |
| 接続準備中 | RTC 接続開始後、接続成立前 |
| 接続済み | control plane が成立している |
| 通話可能 | media を開始できる peer がいる |
| 再接続中 | RTC が復旧中または再接続待機中 |
| 失敗 | 接続、認証、media 開始のいずれかで復旧可能な失敗がある |

UI 表示名や画面遷移は `docs/spec/App/UI` を正とする。

## 異常時

| 事象 | App の扱い |
|---|---|
| invite token 不正 | group へ参加せず、受理失敗として表示する |
| group secret 取得失敗 | 接続を開始せず、credential 不足として表示する |
| RTC 接続失敗 | RTC error を要約して表示し、再試行可能にする |
| peer 認証失敗 | 不正な peer として表示し、音声 media は開始しない |
| media 開始失敗 | 接続失敗とは分けて、音声開始失敗として表示する |
| App data decode 失敗 | 該当 message を無視し、必要なら debug log に残す |

## ログ

| 事象 | Event | Level |
|---|---|---|
| 接続開始 | `rtc.connection.started` | `.info` |
| 接続失敗 | `rtc.connection.failed` | `.error` |
| route handover | `rtc.route.handover.started` | `.notice` |
| route 品質劣化 | `rtc.route.degraded` | `.warning` |
| peer 認証失敗 | `rtc.handshake.rejected` | `.error` |
| App data decode 失敗 | `rtc.application_data.decode_failed` | `.debug` |

metadata と記録禁止情報は `docs/spec/App/ログ仕様.md` を正とする。
