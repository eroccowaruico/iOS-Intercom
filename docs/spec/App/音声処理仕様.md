# RideIntercom 音声処理仕様

## 目的

本書は現行実装における音声入力、VAD、送信、受信、再生、Audio Check を定義する。  
音声処理の背景、意味、意図、状態変化、異常時の考え方を明記し、UI 表示値と内部処理の対応を追えるようにする。

## 適用範囲

| 区分 | 内容 |
|---|---|
| 通話送信 | マイク入力から VAD を経て `OutboundAudioPacket` を生成するまで |
| 通話受信 | 受信 packet の受理、jitter buffer、ミックス、出力まで |
| 診断 | Diagnostics と Settings の音声メーターが参照する値 |
| Audio Check | 5 秒録音 + 5 秒再生 |

## 背景と基本方針

| 項目 | 内容 |
|---|---|
| 背景 | 音楽併用、同時通話、低遅延、電池効率を両立する必要がある |
| 目的 | 話し始めを取りこぼさず、無音時帯域を抑え、受信音声を途切れにくく再生する |
| 方針 | 送信は VAD と pre-roll を使い、受信は jitter buffer と最終ゲイン適用を使う |
| 意図 | 「聞こえる」「話せる」「どこで止まっているか分かる」を同時に満たす |
| 異常系の考え方 | 権限拒否、I/O 変更失敗、codec 不整合などは文言や診断値へ反映し、状態遷移を追えるようにする |

## 音声処理アーキテクチャ

| コンポーネント | 実装 | 役割 | 意味 |
|---|---|---|---|
| Audio session | `AudioSessionManager` | 通話用 / Audio Check 用セッション設定と I/O ルーティング | OS 音声資源の入口 |
| 入力監視 | `SystemAudioInputMonitor` | AVAudioEngine tap、レベル算出、16kHz 変換 | 送信系の入口 |
| VAD / packet 生成 | `AudioTransmissionController` | VAD、pre-roll、keepalive、voice packet 生成 | 送るか送らないかの判断点 |
| 再生バッファ | `JitterBuffer` | playout delay、寿命管理、ready frame 排出 | 受信揺らぎ吸収 |
| ミックス/再生 | `BufferedAudioFramePlayer`, `SystemAudioOutputRenderer` | 出力ゲイン適用後の再生 | 可聴音への最終段 |
| 状態統合 | `IntercomViewModel` | UI 用メーター、状態、エラー文言の公開 | 画面との接続点 |

補足:

| 項目 | 現行から今後の設計方針 | 意味 |
|---|---|---|
| Control Plane | advertise / browse / handshake / metadata/control 送受信 | 音声 I/O を上げなくても維持できる接続面 |
| Media Plane | AVAudioSession, input monitor, packetize, jitter, renderer | 接続確立後にのみ開始する音声面 |
| 分離理由 | 接続待受や metadata 交換のために毎回 mic / speaker を上げない | 接続安定性と音声資源管理を両立する |

## 実行フロー

| ステップ | 入力 | 処理 | 出力 | 意味・意図 |
|---|---|---|---|---|
| Mic Capture | AVAudioEngine input tap | RMS 算出、サンプル抽出、16kHz へ変換 | level, samples | 入力を UI と VAD の両方へ供給する |
| VAD | level | 閾値と noise floor 追従で発話判定 | `idle / attack / talking / release` | 無音時送信抑制と話頭保護の両立 |
| Packetize | frameID, samples, VAD 状態 | pre-roll 付与、keepalive 生成 | `OutboundAudioPacket` | 送信単位へ正規化する |
| Encode | samples | codec 別符号化 | `EncodedVoicePacket` | payload を通信向け表現へ変換する |
| Diagnostics | frameID, media codec, fallback | 送信側のローカル診断 event を生成 | diagnostics event | 実際に送る codec は media payload から取得し、別control payloadでは送らない |
| Transport | packet | Local / Internet へ送信 | 転送データ | 経路へ流す |
| Receive | 受信データ | 復号、デコード、検証 | `ReceivedAudioPacket` | 再生可能な受信表現へ戻す |
| Jitter | packet + timestamp | playout 遅延制御、期限切れ破棄 | Ready frames | 途切れにくさを優先する |
| Output | ready frames | ミックス、ゲイン適用、再生 | スピーカー出力 | 実際に聞こえる音へ変換する |

補足: 上記フローは Media Plane が起動した後にのみ動作する。group 選択、standby、invite、handshake だけでは開始しない。

## Audio Session

| 用途 | category | mode | options | 背景・意味 |
|---|---|---|---|---|
| Intercom 通話 | `playAndRecord` | `default` | `mixWithOthers`, `allowBluetooth`, `allowBluetoothA2DP` | 音楽併用と通話入出力を両立する。接続しただけで他アプリ音量や playback level を変えにくくする。`AVAudioSession` の `duckOthers` は常用しない |
| Audio Check | `playAndRecord` | `default` | `allowBluetooth`, `allowBluetoothA2DP` | 自己診断時は通話最適化より単純な I/O 確認を優先する |

補足: 通話時のみ `mixWithOthers` を付与するため、音楽併用前提は通話経路に対して明示されている。

補足: `AudioSessionManager.configureForIntercom()` は接続開始時ではなく、Media Plane 開始時にだけ呼ぶ想定へ改める。接続済みでも音声未開始の間は `AVAudioSession` を起動しない。

補足: `Duck Other Audio` は iOS の voice processing 系 API で扱う。`AVAudioSession.CategoryOptions.duckOthers` を通話中に動的に付け外しすると、他アプリ音声が duck ではなく停止・中断したように見えるケースがあるため、この方式は採用しない。

補足: `Duck Other Audio` は `Audio Check` 再生には適用しない。macOS では同等 API を使っていないため、当該設定項目自体を表示しない。

補足: `AVAudioSession.mode = .voiceChat` は chat 用の動的処理により接続しただけで playback level が下がることがあるため、現行実装では常用しない。voice processing が必要な機能は `AVAudioInputNode.setVoiceProcessingEnabled(_:)` 側で個別に有効化する。

## 入力仕様

補足: 入力系は Media Plane の構成要素であり、peer 認証完了前には開始しない。

### マイク入力

| 項目 | 値 | 意味・意図 | 異常系の考え方 |
|---|---|---|---|
| 実装 | `AVAudioEngine.inputNode.installTap` | OS の入力を継続監視する | なし |
| `bufferSize` | `128` | 低遅延寄りの入力粒度 | なし |
| レベル算出 | `AudioLevelMeter.rmsLevel(samples:)` | UI と VAD の共通指標に使う | なし |
| リサンプル先 | `16_000 Hz` | 通話系処理の共通サンプルレートへ揃える | なし |
| 権限未確定 | 要求を投げて `microphonePermissionRequestPending` を返す | 権限状態を明示して再試行可能にする | その場で接続続行しない |
| 権限拒否/制限 | `microphonePermissionDenied` を返す | 送信不能理由を明確化する | Privacy 設定変更を促す |
| ミュート時の入力エンジン | エンジンは停止せず `isVoiceProcessingInputMuted = true` で VP 入力を抑止する | ducking 設定がマイクミュートと独立して動作するために入力エンジンを維持する必要がある |

補足: ミュート時にエンジンを停止する方式（mute auto-stop）は廃止する。エンジンを停止すると `voiceProcessingOtherAudioDuckingConfiguration` の更新先が失われ、ダッキングレベルの動的切替ができなくなるため。

### Sound Isolation

| 項目 | 現行仕様 | 背景・意味 |
|---|---|---|
| サポート判定 | `SoundIsolationBackend.isSupported` | OS / 端末差異を吸収する |
| 実装 | `AVAudioInputNode.setVoiceProcessingEnabled` | 入力前処理特性を切り替える |
| 既定値 | `ON` | 走行時の環境雑音抑制を標準挙動とする |
| UI 配置 | `Settings > Audio I/O > Sound Isolation` | 入出力関連設定として `Duck Other Audio` の上に置く |
| 稼働中の切替 | voice processing 状態変更時に入力 node と tap を再構成してから capture を再開する | runtime 切替で入力が消える挙動を避ける |
| 非対応時 | `isSoundIsolationEnabled = false` 固定 | ON/OFF と N/A を明確に分ける |

補足: `Sound Isolation` はマイク入力に対してのみ適用する。受信再生、参加者出力、他アプリ再生音量には直接作用させない。

### Other Audio Ducking

| 項目 | 現行仕様 | 背景・意味 |
|---|---|---|
| サポート判定 | iOS のみ有効。macOS は非対応 | platform ごとの差異を UI/仕様へ反映する |
| 実装 | アプリ側の判定は出力系で行い、iOS 適用点は `AVAudioInputNode.voiceProcessingOtherAudioDuckingConfiguration` を使う | 設定判断は output concern、OS 適用 API は platform 制約として input node にある |
| `enableAdvancedDucking` | 常に `true` | `false` は standard ducking（voice processing が有効な間は常時ダッキング）となり、接続しただけで他アプリ音量が下がる。`true` は voice activity ベースのダッキングとなり、音声アクティビティがある時だけ発動する |
| `duckingLevel` 制御 | `Duck Other Audio` が ON かつアプリ側の発動条件を満たす時は `.default`、それ以外は `.min` | アプリ側の判定で強度を切り替える。`.min` は voice activity ベースでも最小限の影響に抑える |
| アプリ側の発動条件 | `Duck Other Audio` が ON かつ直近 1.0 秒以内に可聴な受信音声 frame を post-gain で再生へ渡した時のみ | 接続しただけや自分の送話だけでは他アプリ音量へ影響を与えない |
| マイクミュート時の動作 | ダッキングはマイクミュート状態と独立して動作する。入力エンジンは停止せず `isVoiceProcessingInputMuted` で入力だけを抑止するため、受信音声の有無に応じて `duckingLevel` を随時変更できる | マイク OFF でも受信音声の再生時にダッキング強度を切り替えたい要件に対応する |
| ducking 設定の適用方式 | `voiceProcessingOtherAudioDuckingConfiguration` のプロパティ更新のみ。エンジン再起動は行わない | 動的な切替で他アプリ音声の停止・中断を防ぐ |
| voice processing 依存 | ducking 使用時は `setVoiceProcessingEnabled(true)` を前提とする | 入力ノードの ducking API が voice processing 前提のため |
| Sound Isolation OFF 時 | `isVoiceProcessingBypassed = true` とし、ducking は維持する | ducking だけ必要で uplink 処理は弱めたいケースを許容する |
| 採用理由 | `AVAudioSession.duckOthers` の動的再適用をやめる | 再生開始時に他アプリ音声が stop / interrupt される挙動を避ける |

補足: `Duck Other Audio` の発動条件は受信音声のみである。ローカル発話、VAD の talking 状態、接続済みだが受信がない状態はダック条件に含めない。

補足: 受信音声があっても、`Output mute` が ON、`masterOutputVolume == 0`、または当該 peer の `remoteOutputVolume == 0` で実質再生されない場合はダック条件に含めない。

補足: ダック判定は jitter buffer から取り出した raw frame ではなく、master / participant volume と mute を反映した post-gain frame を renderer へ渡した時点で更新する。無音 frame、空 frame、再生へ渡らなかった frame はダック条件に含めない。

補足: `enableAdvancedDucking: true` は local と remote 双方の voice activity に反応するため、ローカル発話時にも iOS 側が最小限のダッキングを適用する可能性がある。アプリ側で `duckingLevel` を `.min` にすることで影響を抑えるが、完全にゼロにはできない。これは voice processing ベースの ducking API のプラットフォーム制約である。

## VAD 仕様

| 項目 | 値 | 意味 |
|---|---|---|
| 状態 | `idle`, `attack`, `talking`, `release` | 発話判定の段階 |
| 既定閾値 | `0.003` | 通常感度 |
| 最小閾値 | `0.00025` | 感度下限 |
| 最大閾値 | `0.12` | 感度上限 |
| `attackFrames` | `1` | 話し始め反応速度 |
| `releaseFrames` | `40` | 話し終わり保持 |
| noise floor 初期値 | `minThreshold` | 環境雑音基準の起点 |

補足:

| 項目 | 現行仕様 | 意味・意図 |
|---|---|---|
| start 判定 | `noiseFloor + max(threshold, noiseFloor * 0.6)` を下限付きで使用 | 静かな環境でも騒がしい環境でも誤検出を抑える |
| release 判定 | `noiseFloor + max(threshold * 0.35, noiseFloor * 0.2)` を下限付きで使用 | 語尾切れを抑える |
| noise floor 更新 | state ごとに smoothing を変える | 環境変化へ追従しつつ過反応しない |

## 送信仕様

### フレーム処理

| 項目 | 仕様 | 意味・意図 |
|---|---|---|
| フレーム ID | `nextAudioFrameID` を 1 から採番 | 処理順序と診断追跡の基準にする |
| ミュート中 | ローカルメーターを 0 にし、voice packet を出さない | UI と送信実態を一致させる |
| packet 生成 | `audioTransmissionController.process(frameID:level:samples:)` | VAD 判定と packet 化の統合入口 |
| 発話状態 | 生成 packet に `voice` が含まれるかで `isVoiceActive` を更新 | UI の talking 状態と送信実体を対応させる |

### packet 生成条件

| 項目 | 値 | 意味 |
|---|---|---|
| pre-roll 保持数 | `20 frames` | 話し始め頭欠けを防ぐ |
| keepalive 間隔 | `50 frames` | 無音時も経路維持と品質観測を可能にする |
| voice packet | VAD 通過時に生成 | 実音声送信 |
| keepalive | 無音時の維持 packet | 送話なしでも接続情報を流す |

### codec

| 項目 | 現行仕様 | 背景・意味 |
|---|---|---|
| UI 表示 | PCM 16-bit / HE-AAC v2 VBR / Opus | 送信方式の選択概念を UI 上で持つ |
| 実送信 | `setPreferredTransmitCodec(_:)` が常に `.pcm16` を設定するため PCM 固定 | 実送信系は単一 codec として振る舞う |
| HE-AAC 品質 | setter が常に `.medium` を設定する | 品質概念は残るが送信実体と分離される |
| Diagnostics metadata | requested codec はアプリ設定、media codec は実際の media payload、fallback は送信側診断として保持する | UI 選択概念と実送信実態の差を診断可能にする |

## 受信仕様

### 受信受理

| 項目 | 仕様 | 意味・意図 | 異常系の考え方 |
|---|---|---|---|
| group 一致 | `ReceivedAudioPacketFilter` で確認 | 他グループ音声を排除する | 不一致は破棄 |
| Local 認証 | `authenticatedPeerIDs` にない peer は破棄 | 接続と音声受理を分離する | 未認証音声は再生しない |
| decode 成功時のみ | 参加者メーター、RX/JIT/PLAY カウンタを更新 | UI を「実データに基づく状態」に限定する | 接続イベントだけでメーターを上げない |
| keepalive | 再生しない | 維持信号と音声本体を分離する | 無音として扱う |

### jitter buffer

| 項目 | 値 | 意味 |
|---|---|---|
| playout delay | `0.015 sec` | 低遅延と安定再生のバランス |
| packet lifetime | `2.0 sec` | 古い音を再生しない寿命 |
| drain 周期 | `RepeatingCallTicker(interval: 10ms)` 上で `handleCallTick` が実行 | 小刻みに ready frame を取り出す |

### 参加者状態反映

| 項目 | 仕様 | 意味 |
|---|---|---|
| `receivedAudioPacketCount` | デコード済み音声 packet を受理したとき増加 | 受信実績 |
| `queuedAudioFrameCount` | jitter buffer へ enqueue したとき増加 | 再生待ち実績 |
| `playedAudioFrameCount` | ready frame として drain したとき増加 | 再生実績 |
| `voiceLevel`, `voicePeakLevel` | デコード済みサンプルから算出 | 相手入力メーター表示値 |
| `isTalking` | 音声受信で `true`、`0.6 sec` 無音受信で `false` | 話中表示の維持と解除 |

### 参加者音声状態の意味

| 状態 | 判定の意味 |
|---|---|
| `Audio Idle` | まだデコード済み音声受理、queue、play の実績がない |
| `Received` | 受信はしたが、まだ queue や play に進んでいない |
| `Receiving` | jitter buffer に滞留しており、再生待ちがある |
| `Playing` | 再生対象として drain され、実際に出力へ回っている |

## 出力仕様

補足: 出力 renderer / jitter / mixer も Media Plane の構成要素であり、接続済みだが音声未開始の状態では動作させない。

背景: OS のマスターボリュームは他アプリ再生音にも共通で作用するため、RideIntercom だけを相対的に大きくしたい場面がある。通話画面のマスター出力は、OS 音量を上げずにインターコム音声だけを持ち上げられるよう、通常最大をスライダー中央値へ置き、その上側をブースト領域として扱う。

### 再生パス

| 項目 | 仕様 | 意味 |
|---|---|---|
| 出力 renderer | `SystemAudioOutputRenderer` | 最終出力の実体 |
| サンプルレート | `16_000 Hz` | 再生系の共通レート |
| チャンネル数 | `1 ch` | 出力ミックス後の単一系統 |
| 再生 | `AVAudioPlayerNode.scheduleBuffer` | バッファ単位で再生へ回す |

### 出力ゲイン

| 項目 | 値 | 意味 |
|---|---|---|
| master output volume | `0.0 ... 2.0` | 全体最終出力ゲイン |
| normal 値 | `1.0` | 通常最大 |
| remote output volume | peer ごとに `0.0 ... 1.0` | 参加者別の個別調整 |
| output mute | 最終ゲインを 0 とする | 再生のみ停止 |
| soft clip | 閾値 `0.9` まで線形、超過分を指数カーブで `1.0` に漸近させる | boost 時の過大値を滑らかに抑制しつつ通常域の音量損失を排除する |

### マスター出力表示仕様

| 表示箇所 | 参照元 | 仕様 | 意味 |
|---|---|---|---|
| Settings > Audio Check > Speaker Output meter | `diagnosticsOutputLevel / diagnosticsOutputPeakLevel` | Audio Check 再生中は Audio Check の再生出力、通常通話中はマスター出力として実際に schedule した最終出力 RMS / Peak を表示する | 何を聞かせたかではなく、何を再生へ回したかを示す |
| Diagnostics > Playback Summary | `snapshot.playback.summary` | ミックス後かつ master / remote / mute 適用後の最終出力 RMS、および schedule batch / frame 数を表示する | 再生段まで到達したかを見る |
| Participants の各行メーター | 各 remote member の状態値 | 参加者ごとの状態確認用であり、マスター出力表示ではない | 参加者入力と最終出力を混同しない |

## Audio Check

### フロー

| ステップ | 内容 | 時間 | 意味・意図 |
|---|---|---|---|
| `recording` | マイク入力を記録し、入力メーターを更新 | 5 秒 | 入力取得の確認 |
| `playing` | 録音サンプルを再生し、出力メーターを更新 | 5 秒 | 再生経路確認 |
| `completed` | 完了文言へ更新 | 即時 | 正常終了 |
| `failed` | セットアップ失敗またはサンプル空 | 即時 | 失敗理由の明示 |

### 現行仕様

| 項目 | 仕様 | 意味 |
|---|---|---|
| 開始条件 | `audioCheckPhase` が `recording` / `playing` でない | 多重実行防止 |
| 所有権 | 通話未開始時のみ `audioCheckOwnsAudioPipeline = true` | 通話中と診断中の所有関係を分ける |
| 録音失敗条件 | サンプル 0 件 | 入力不成立を明確化 |
| 再生 codec 表示 | 文言上は `PCM 16-bit` | 画面上の理解を単純化する |
| 実際の再生 codec | `finishAudioCheckRecording` 内で `playbackCodec = .pcm16` 固定 | 再生経路は単一 codec で確認する |

## ミュート仕様

| 項目 | 仕様 | 意味 |
|---|---|---|
| ローカルミュート | `toggleMute()` | 送話停止の主操作 |
| 2 秒継続時 | `audioInputMonitor.stop()` し `isMicrophoneCaptureSuspendedByMute = true` | 不要入力処理を抑える |
| 解除時 | `audioInputMonitor.start()` を試行 | 送話再開 |
| peer 同期 | RideIntercom application data として mute state を送信 | 相手画面と送話状態を整合させる |

## 異常時仕様

| 事象 | 表示/動作 | 考え方 |
|---|---|---|
| マイク権限未決定 | `Microphone permission requested. Allow access, then connect again.` | 権限要求待ちを明示し再試行可能にする |
| マイク権限拒否 | `Microphone access is off. Enable it in Privacy & Security, then connect again.` | 設定変更を促す |
| 入力デバイス変更失敗 | `Audio input device change failed` | I/O 問題を通話失敗と区別して示す |
| 出力デバイス変更失敗 | `Audio output device change failed` | 再生先問題を分離して示す |
| それ以外の音声起動失敗 | `Audio setup failed` | 詳細不明でも音声系失敗として扱う |
