# RideIntercom App 設定値一覧

## 目的

本書は RideIntercom App 本体が画面から設定する値だけを整理する。

画面から直接設定しない library 設定、固定値、画面設定から導出して package へ渡す値は `docs/spec/App/setting parameters/packages` を正とする。package 内部の既定値、範囲、Audio Unit parameter、routing の詳細は `docs/spec/packages` を正とする。

## 設定の責務境界

| 領域 | App 本体に残すもの | package 側へ寄せるもの |
|---|---|---|
| ユーザーとグループ | local member、display name、group list、group secret の保存と選択 | なし |
| Audio Session / device | 入出力デバイスの選択状態、UI 表示、再適用タイミング | `SessionManager` が category / mode / option、OS 差分、snapshot、device selection を扱う |
| 入力処理 | Sound Isolation ON/OFF、他音声 ducking のユーザー opt-in、マイクミュート | `SessionManager` と `VADGate` が voice processing、advanced ducking、VAD / gate の詳細を扱う |
| ミキシング / 再生 | マスター音量、出力ミュート、参加者別音量 | `AudioMixer` が bus、effect chain、format、routing を扱う |
| Effectors | App が使う effect の有効/無効と挿入位置 | `SoundIsolation`、`DynamicsProcessor`、`PeakLimiter` が各 effect の設定値と Audio Unit 適用を扱う |
| RTC | group credential、local peer、route policy、App data namespace | `RTC` が接続、handover、packet audio、WebRTC adapter、media lifecycle を扱う |

## App 本体に残すユーザー設定

| 設定名 | App 側識別子 | 型 | 既定値 | 範囲/候補 | 反映タイミング | 関連 package |
|---|---|---|---|---|---|---|
| 入力デバイス | `selectedInputPort` | device selection | system default | system default と利用可能入力 | audio session 構成済みなら即時。それ以外は次回 media 開始時 | SessionManager |
| 出力デバイス | `selectedOutputPort` | device selection | system default | iOS: system default / speaker。macOS: system default と利用可能出力 | audio session 構成済みなら再構成。再生中は出力 renderer / mixer も再起動 | SessionManager / Mixer |
| Sound Isolation | `isSoundIsolationEnabled` | `Bool` | `true` | `true/false` | 入力処理中は可能な限り即時。未対応時は `false` に丸める | SessionManager |
| 他音声ダック | `isDuckOthersEnabled` | `Bool` | `false` | `true/false` | ユーザー設定は即時。実際の ducking は可聴な受信音声がある間だけ有効 | SessionManager |
| VAD 感度 | `voiceActivityDetectionThreshold` | `Float` | `0.003` | `0.00025...0.12` | 即時 | VADGate |
| マスター出力音量 | `masterOutputVolume` | `Float` | `1.0` | `0.0...2.0` | 即時 | Mixer |
| マスター出力ミュート | `isOutputMuted` | `Bool` | `false` | `true/false` | 即時 | Mixer |
| 参加者別出力音量 | `remoteOutputVolumes[peerID]` | `[PeerID: Float]` | 未設定時 `1.0` | `0.0...1.0` | 即時 | Mixer |
| ローカルマイクミュート | `isMuted` | `Bool` | `false` | `true/false` | 即時。入力エンジンは停止しない | SessionManager / RTC |
| Audio Check 実行 | `startAudioCheck()` | Action | なし | 実行のみ | ボタン押下時 | SessionManager |
| 設定リセット | `resetAllSettings()` | Action | なし | 実行のみ | ボタン押下時 | App |

## 永続化される値

| 項目 | 保存先 | 備考 |
|---|---|---|
| `RideIntercom.localMemberID` | `UserDefaults` | 初回生成後に保持 |
| `RideIntercom.localDisplayName` | `UserDefaults` | 初回は `You` |
| グループ一覧 | `UserDefaults` | `RideIntercom.groups`。`accessSecret` は保存しない |
| グループ secret | Keychain | `RideIntercom.GroupCredential` サービス |

## 永続化されない値

| 項目 | 理由 |
|---|---|
| 入出力デバイス選択 | OS route と利用可能デバイスが起動ごとに変わるため |
| Sound Isolation | 実行環境の対応状況に依存するため |
| 他音声ダック | 通話中の一時的な再生状態に依存するため |
| VAD 感度 | 現状は調整用の実行時設定として扱うため |
| 出力音量 / ミュート | 通話中の実行時 gain として扱うため |
| 参加者別出力音量 | peer の参加状態に依存するため |
| Audio Check 状態 | 実行時状態のみ |

## 設定リセット対象

| 項目 | リセット後の値 |
|---|---|
| 入力デバイス | system default |
| 出力デバイス | system default |
| Sound Isolation | `true` |
| 他音声ダック | `false` |
| VAD 感度 | `0.003` |
| マスター出力音量 | `1.0` |
| マスター出力ミュート | `false` |
| 参加者別出力音量 | 未設定扱い |

`resetAllSettings()` は local member、グループ一覧、参加者構成、group secret、接続中の group 選択を変更しない。
