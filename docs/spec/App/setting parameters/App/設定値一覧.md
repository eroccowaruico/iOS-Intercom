# RideIntercom App 設定値一覧

## 目的

本書は RideIntercom App 本体が保持する設定値と、`RideIntercom/packages` の各 package を使うときに App から渡す設定だけを整理する。

package 内部の既定値、範囲、Audio Unit parameter、routing の詳細は `docs/spec/packages` を正とする。本書ではそれらを再定義せず、App が UI、永続化、実行時ポリシーとして持つ値だけを扱う。

## 設定の責務境界

| 領域 | App 本体に残すもの | package 側へ寄せるもの |
|---|---|---|
| ユーザーとグループ | local member、display name、group list、group secret の保存と選択 | なし |
| Audio Session / device | 入出力デバイスの選択状態、UI 表示、再適用タイミング | `SessionManager` が category / mode / option、OS 差分、snapshot、device selection を扱う |
| 入力処理 | Sound Isolation ON/OFF、他音声 ducking のユーザー opt-in、マイクミュート | `SessionManager` と `VADGate` が voice processing、advanced ducking、VAD / gate の詳細を扱う |
| ミキシング / 再生 | マスター音量、出力ミュート、参加者別音量 | `AudioMixer` が bus、effect chain、format、routing を扱う |
| Effectors | App が使う effect の有効/無効と挿入位置 | `SoundIsolation`、`DynamicsProcessor`、`PeakLimiter` が各 effect の設定値と Audio Unit 適用を扱う |
| RTC | group credential、local peer、route policy、App data namespace | `RTC` が接続、handover、packet audio、WebRTC adapter、media lifecycle を扱う |

## App 本体に残すユーザー設定

| 設定名 | App 側識別子 | 型 | 既定値 | 範囲/候補 | 反映タイミング | package 連携 |
|---|---|---|---|---|---|---|
| 入力デバイス | `selectedInputPort` | device selection | system default | system default と利用可能入力 | audio session 構成済みなら即時。それ以外は次回 media 開始時 | `AudioSessionConfiguration.preferredInput` |
| 出力デバイス | `selectedOutputPort` | device selection | system default | iOS: system default / speaker。macOS: system default と利用可能出力 | audio session 構成済みなら再構成。再生中は出力 renderer / mixer も再起動 | `AudioSessionConfiguration.preferredOutput`、speaker 選択時は `defaultToSpeaker = true` |
| Sound Isolation | `isSoundIsolationEnabled` | `Bool` | `true` | `true/false` | 入力処理中は可能な限り即時。未対応時は `false` に丸める | `AudioInputVoiceProcessingConfiguration.soundIsolationEnabled` |
| 他音声ダック | `isDuckOthersEnabled` | `Bool` | `false` | `true/false` | ユーザー設定は即時。実際の ducking は可聴な受信音声がある間だけ有効 | `AudioInputVoiceProcessingConfiguration.otherAudioDuckingEnabled` と `duckingLevel` |
| VAD 感度 | `voiceActivityDetectionThreshold` | `Float` | `0.003` | `0.00025...0.12` | 即時 | `VADGateConfiguration` への変換元。VAD の attack/release 等は package 仕様を正とする |
| マスター出力音量 | `masterOutputVolume` | `Float` | `1.0` | `0.0...2.0` | 即時 | `AudioMixer` の master bus volume、または RTC の output volume API へ反映 |
| マスター出力ミュート | `isOutputMuted` | `Bool` | `false` | `true/false` | 即時 | master bus volume を `0` 扱い、または RTC の output mute API へ反映 |
| 参加者別出力音量 | `remoteOutputVolumes[peerID]` | `[PeerID: Float]` | 未設定時 `1.0` | `0.0...1.0` | 即時 | peer bus volume、または RTC の remote output volume API へ反映 |
| ローカルマイクミュート | `isMuted` | `Bool` | `false` | `true/false` | 即時。入力エンジンは停止しない | `AudioInputVoiceProcessingConfiguration.inputMuted` と RTC App data |
| Audio Check 実行 | `startAudioCheck()` | Action | なし | 実行のみ | ボタン押下時 | `SessionManager` と audio pipeline を一時利用 |
| 設定リセット | `resetAllSettings()` | Action | なし | 実行のみ | ボタン押下時 | App 所有の非永続設定だけを初期化 |

## package 使用時に App から渡す設定

### SessionManager

| package 設定 | App から渡す値 | 備考 |
|---|---|---|
| `AudioSessionConfiguration.mode` | `.default` | `voiceChat` は App が明示的な通話モード設定を持つまで使わない |
| `AudioSessionConfiguration.defaultToSpeaker` | `selectedOutputPort` が built-in speaker のとき `true` | 出力 UI の選択から導出する |
| `AudioSessionConfiguration.prefersEchoCancelledInput` | `false` | Echo cancellation を App 設定として出すまでは固定 |
| `AudioSessionConfiguration.preferredInput` | `selectedInputPort` から変換 | system default / device ID を package 型へ変換する |
| `AudioSessionConfiguration.preferredOutput` | `selectedOutputPort` から変換 | iOS speaker、macOS device ID の差分は package 側で扱う |
| `AudioInputVoiceProcessingConfiguration.soundIsolationEnabled` | `isSoundIsolationEnabled` | 未対応環境では package の失敗または snapshot を見て App 表示を `false` / N/A にする |
| `AudioInputVoiceProcessingConfiguration.otherAudioDuckingEnabled` | `isDuckOthersEnabled && isOtherAudioDuckingActive` | `isOtherAudioDuckingActive` は受信音声が可聴で、Audio Check 中ではない場合だけ `true` |
| `AudioInputVoiceProcessingConfiguration.duckingLevel` | active 時 `.normal`、それ以外 `.minimum` | 他音声 ducking OFF でも voice processing を維持する場合は `.minimum` を渡す |
| `AudioInputVoiceProcessingConfiguration.inputMuted` | `isMuted` | ミュート時も入力ノードと voice processing の更新先を維持する |

### RTC

| package 設定 | App から渡す値 | 備考 |
|---|---|---|
| `CallRouteConfiguration.enabledRoutes` | 現状は `[.multipeer]` | WebRTC を UI / credential / signaling と合わせて有効化するまでは local route のみ |
| `CallRouteConfiguration.preferredRoute` | `.multipeer` | 近距離・オフライン動作を優先する |
| `CallRouteConfiguration.selectionMode` | `.singleRoute` | WebRTC を有効化するまでは自動 fallback / restore を使わない |
| `AudioFormatDescriptor` | App の capture / packet audio format から生成 | sample rate や channel count の定数は App に重複定義せず、capture adapter 境界で変換する |
| `RTCCredential` | group secret から生成 | secret は Keychain に保存し、group list には保存しない |
| `PeerDescriptor` | local member ID / display name から生成 | 表示名の永続化は App の責務 |
| `ApplicationDataMessage.namespace` | `rideintercom.peerMuteState` | namespace ごとの payload schema は App 側で管理する |

### AudioMixer / Effectors

| package 設定 | App から渡す値 | 備考 |
|---|---|---|
| `AudioMixer` format | package default、または capture / output adapter が要求する format | format の既定値は `docs/spec/packages/Audio/Mixer.md` を正とする |
| peer bus volume | `remoteOutputVolumes[peerID]` | 未設定 peer は `1.0` |
| master bus volume | `isOutputMuted ? 0 : masterOutputVolume` | boost 上限は App UI の `2.0` に限定する |
| `VADGateConfiguration` | 既定値、または `voiceActivityDetectionThreshold` から導出した App preset | attack/release、noise floor、gain ramp の詳細は `docs/spec/packages/Audio/Effectors/VADGate.md` を正とする |
| `VoiceIsolationConfiguration` | effect chain に挿入する場合は package default | 入力 node の voice processing は `SessionManager` 側を優先する |
| `DynamicsProcessorConfiguration` | package default | App UI で詳細調整を出すまでは App 設定にしない |
| `PeakLimiterConfiguration` | package default | App UI で詳細調整を出すまでは App 設定にしない |

## 永続化される値

| 項目 | 保存先 | 備考 |
|---|---|---|
| `RideIntercom.localMemberID` | `UserDefaults` | 初回生成後に保持 |
| `RideIntercom.localDisplayName` | `UserDefaults` | 初回は `You` |
| グループ一覧 | `UserDefaults` | `RideIntercom.groups`。`accessSecret` は保存しない |
| グループ secret | Keychain | `RideIntercom.GroupCredential` サービス |

## 永続化されない値

| 項目 | 理由 |
|---|---|
| 入出力デバイス選択 | OS route と利用可能デバイスが起動ごとに変わるため |
| Sound Isolation | 実行環境の対応状況に依存するため |
| 他音声ダック | 通話中の一時的な再生状態に依存するため |
| VAD 感度 | 現状は調整用の実行時設定として扱うため |
| 出力音量 / ミュート | 通話中の実行時 gain として扱うため |
| 参加者別出力音量 | peer の参加状態に依存するため |
| Audio Check 状態 | 実行時状態のみ |

## 設定リセット対象

| 項目 | リセット後の値 |
|---|---|
| 入力デバイス | system default |
| 出力デバイス | system default |
| Sound Isolation | `true` |
| 他音声ダック | `false` |
| VAD 感度 | `0.003` |
| マスター出力音量 | `1.0` |
| マスター出力ミュート | `false` |
| 参加者別出力音量 | 未設定扱い |

`resetAllSettings()` は local member、グループ一覧、参加者構成、group secret、接続中の group 選択を変更しない。

## App 本体から外す設定値

次の値は App の設定一覧には残さず、package 仕様または package adapter の内部値として扱う。

| 領域 | App から外す値 | 参照先 |
|---|---|---|
| VAD / gate | attack、release、hangover、noise floor、gate gain、pre-roll | `docs/spec/packages/Audio/Effectors/VADGate.md`、`docs/spec/packages/RTC.md` |
| Jitter / packet audio | playout delay、packet lifetime、sequence / duplicate filter | `docs/spec/packages/RTC.md` |
| Audio Session | iOS category options、CoreAudio device 切替、advanced ducking の OS API 詳細 | `docs/spec/packages/Audio/SessionManager.md` |
| AudioMixer | default format、bus routing、effect chain 再構築、effect index | `docs/spec/packages/Audio/Mixer.md` |
| Effectors | Audio Unit parameter、support check、fallback policy | `docs/spec/packages/Audio/Effectors/*.md` |
| codec 詳細 | PCM packet 化、WebRTC route-managed media、codec fallback | `docs/spec/packages/RTC.md` |
| disabled codec UI | `preferredTransmitCodec`、`heAACv2Quality` | App 設定としては残さず、RTC の codec policy へ寄せる |
