# RideIntercom App 設定値一覧

## 目的

本書は、作り直す RideIntercom App がユーザー操作として保持する設定値だけを定義する。

画面から直接設定しない library 設定、固定値、画面設定から導出して package へ渡す値は `docs/spec/App/setting parameters/packages` を正とする。package 内部の既定値、範囲、Audio Unit parameter、routing の詳細は `docs/spec/packages` を正とする。

## 設定の責務境界

| 領域 | App 本体に残すもの | package 側へ寄せるもの |
|---|---|---|
| ユーザーとグループ | local member、display name、group list、group secret の保存と選択 | なし |
| Audio Session / device | Session mode、speaker 出力希望、echo cancellation 希望、Duck Other Audio の opt-in、入出力デバイスの選択状態、UI 表示、再適用タイミング | `SessionManager` が category / mode / option、OS 差分、snapshot、device selection、operation report、runtime event を扱う |
| 入力 stream | マイク権限、ローカルミュート | `SessionManager.AudioInputStreamCapture` が input capture、voice processing 初期適用、実行中更新、非対応 no-op を扱う |
| VAD / gate | 感度プリセットの選択、発話表示への採用 | `VADGate` が RMS、ノイズフロア、attack/release、gate gain を扱う |
| ミキシング / 再生 | マスター音量、出力ミュート、参加者別音量、受信 master bus / peer bus の effect 有効状態 | `AudioMixer` が bus、route、effect chain、snapshot を扱う |
| Effectors | 送信 / 受信 master / 参加者別 Sound Isolation effect の有効/無効、App が採用する固定 chain と fallback 表示 | `SoundIsolation`、`DynamicsProcessor`、`PeakLimiter` が Audio Unit 生成、設定正規化、parameter 適用を扱う |
| RTC / Codec | group credential、local peer、route opt-out、route policy、App data namespace、送信 codec のユーザー希望 | `RTC` が接続、handover、packet audio、WebRTC adapter、media lifecycle、metrics を扱い、`Codec` が encode / decode と availability を扱う |
| ログ | ログ最小レベル、構造化 metadata の組み立て | `swift-log` の `Logger` / `LogHandler` が出力先を扱う |

## App 本体に残すユーザー設定

| 設定名 | App 側識別子 | 型 | 既定値 | 範囲/候補 | 反映タイミング | 関連 package |
|---|---|---|---|---|---|---|
| Audio Session mode / Echo Cancellation | `audioSessionProfile` | enum | `.echoCancelledInput` | 画面選択肢は `.standard`、`.voiceChat`。`.echoCancelledInput` は `Burst mode` で echo cancellation を要求する内部値。`.speakerDefault` は旧値互換のみ | media 中は `AudioSessionManager.configure(_:)` を再実行。未開始時は次回 media 開始時 | SessionManager |
| 入力デバイス | `selectedInputDevice` | `AudioSessionDeviceSelection` | `.systemDefault` | system default と `AudioSessionSnapshot.availableInputs` | media 中は `AudioSessionManager.configure(_:)` を再実行。未開始時は次回 media 開始時 | SessionManager |
| スピーカー出力 / 出力デバイス | `selectedOutputDevice` | `AudioSessionDeviceSelection` | `.systemDefault` | Settings では `Use Speaker` を主操作とし、iOS: system default / built-in speaker / built-in receiver。macOS: system default と `AudioSessionSnapshot.availableOutputs` | media 中は session / output renderer / mixer の状態を確認して再構成。未開始時は次回 media 開始時。`Use Speaker` は `defaultToSpeaker` / `preferredOutput` として渡し、echo cancellation の実効値は SessionManager の resolved report を正とする | SessionManager / Mixer |
| 送信 Voice Isolation Effect | `isSoundIsolationEnabled` | `Bool` | `true` | `true/false` | 送信用 SoundIsolation effect chain の有効/無効として扱う。SessionManager の `soundIsolationEnabled` へは渡さない | SoundIsolation / Mixer |
| Duck Other Audio | `isDuckOthersEnabled` | `Bool` | `true` | `true/false` | Audio Session 設定としてユーザー希望を受け取る。実効状態は可聴な受信出力の有無から導出し `updateVoiceProcessing(_:)` へ渡す | SessionManager |
| RTC 有効 route | `enabledRTCTransportRoutes` | `Set<RTC.RouteKind>` | `[.multipeer, .webRTC]` | `.multipeer` と `.webRTC` を既定で opt-in する。空集合を許可し、通信経路をすべて opt-out できる | Settings 変更時に `CallSession.setEnabledRoutes(_:)` へ即時反映し、active RTC connection は停止する。次回接続要求は有効 route だけを使い、空集合では RTC package の `noEnabledRoute` に委ねる | RTC |
| VAD 感度 | `vadSensitivity` | enum | `.standard` | `.lowNoise`、`.standard`、`.noisy` | `VADGate.apply(configuration:)` で即時 | VADGate |
| 送信コーデック | `preferredTransmitCodec` | `CodecIdentifier` | `.mpeg4AACELDv2` | `.pcm16`、`.mpeg4AACELDv2`、`.opus` | 次回送信 frame から。route が未対応または codec unavailable の場合は fallback | Codec / RTC |
| AAC-ELD v2 bitrate | `aacELDv2BitRate` | `Int` | `32_000` | `12_000...128_000` | `preferredTransmitCodec == .mpeg4AACELDv2` の送信 encoder 再構成時 | Codec |
| Opus bitrate | `opusBitRate` | `Int` | `32_000` | `6_000...128_000` | `preferredTransmitCodec == .opus` の送信 encoder 再構成時 | Codec |
| マスター出力音量 | `masterOutputVolume` | `Float` | `1.0` | `0.0...2.0` | `MixerBus.volume` へ即時 | Mixer |
| マスター出力ミュート | `isOutputMuted` | `Bool` | `false` | `true/false` | final output bus の volume 導出へ即時 | Mixer |
| マスター出力 Voice Isolation Effect | `receiveMasterSoundIsolationEnabled` | `Bool` | `false` | `true/false` | RX master bus の SoundIsolation effect chain へ即時 | SoundIsolation / Mixer |
| 参加者別出力音量 | `remoteOutputVolumes[peerID]` | `[PeerID: Float]` | 未設定時 `1.0` | `0.0...1.0` | peer bus の volume へ即時 | Mixer |
| 参加者別 Voice Isolation Effect | `remoteSoundIsolationEnabled[peerID]` | `[PeerID: Bool]` | 未設定時 `false` | `true/false` | peer bus の SoundIsolation effect chain へ即時 | SoundIsolation / Mixer |
| ローカルマイクミュート | `isMuted` | `Bool` | `false` | `true/false` | `AudioInputStreamCapture.updateVoiceProcessing(_:)` で即時。入力 stream は停止しない | SessionManager / RTC |
| Audio Check 実行 | `startAudioCheck()` | Action | なし | 実行のみ | ボタン押下時。通話 media 未開始時だけ実行 | SessionManager / Mixer |
| 設定リセット | `resetAllSettings()` | Action | なし | 実行のみ | ボタン押下時 | App |

## App 本体に残さない設定

| 設定 | 理由 | 参照先 |
|---|---|---|
| codec payload envelope 詳細 | App は送信 codec 希望だけを持ち、payload の構造は解釈しない | `docs/spec/packages/Audio/Codec.md` |
| codec availability 判定の内部詳細 | App は利用可否と fallback 結果だけを表示する | `docs/spec/App/setting parameters/packages/Audio/Codec.md` |
| Audio Session category / mode / options の個別指定 | App は profile だけを持ち、category / mode / option の解決は SessionManager を正とする | `docs/spec/packages/Audio/SessionManager.md` |
| VADGate の全パラメータ | App UI は感度プリセットだけを持ち、attack/release 等は package adapter が決める | `docs/spec/App/setting parameters/packages/Audio/Effectors/VADGate.md` |
| Effectors の Audio Unit parameter | App UI では直接露出せず、採用 chain と fallback だけを扱う | `docs/spec/App/setting parameters/packages/Audio/Effectors/*.md` |
| route fallback / restore policy | package adapter の固定 policy とする | `docs/spec/App/setting parameters/packages/RTC.md` |
| WebRTC route の有効化 UI | Settings の `Internet` toggle で扱う | `docs/spec/App/setting parameters/packages/RTC.md` |

## 永続化される値

| 項目 | 保存先 | 備考 |
|---|---|---|
| `RideIntercom.localMemberID` | `UserDefaults` | 初回生成後に保持 |
| `RideIntercom.localDisplayName` | `UserDefaults` | 初回は `You` |
| グループ一覧 | `UserDefaults` | `RideIntercom.groups`。`accessSecret` は保存しない |
| グループ secret | Keychain | `RideIntercom.GroupCredential` サービス |
| `audioSessionProfile` | `UserDefaults` | Settings の mode と `Burst mode` の echo cancellation 希望を保持する。speaker 出力希望は OS route 依存のため保持しない |
| `vadSensitivity` | `UserDefaults` | 端末の騒音環境に合わせた調整として保持する |
| `preferredTransmitCodec` | `UserDefaults` | ユーザーが選んだ送信 codec 希望 |
| `aacELDv2BitRate` | `UserDefaults` | AAC-ELD v2 選択時の送信 bitrate |
| `opusBitRate` | `UserDefaults` | Opus 選択時の送信 bitrate |
| `enabledRTCTransportRoutes` | `UserDefaults` | RTC package に渡す enabled route のユーザー希望。空配列は全 route opt-out として保持する |

App 実装では `UserDefaultsAppSettingsStore` が上記 6 項目を読み書きする。テストや Preview は `InMemoryAppSettingsStore` を使い、同じ `AppSettingsStoring` 境界で差し替える。

## 永続化されない値

| 項目 | 理由 |
|---|---|
| 入出力デバイス選択 | OS route と利用可能デバイスが起動ごとに変わるため |
| 送信 Voice Isolation Effect | 実行環境の対応状況に依存するため |
| Duck Other Audio | 起動ごとに既定 ON から開始し、実効状態は通話中の一時的な再生状態に依存するため |
| 出力音量 / ミュート | 通話中の実行時 gain として扱うため |
| 参加者別出力音量 | peer の参加状態に依存するため |
| マスター出力 Voice Isolation Effect | 通話中の受信 master bus effect 設定として扱うため |
| 参加者別 Voice Isolation Effect | peer の参加状態に依存するため |
| Audio Check 状態 | 実行時状態のみ |
| runtime report / event | Diagnostics の現在値として保持し、永続化しない |

## 設定リセット対象

| 項目 | リセット後の値 |
|---|---|
| Audio Session mode / Echo Cancellation | `.echoCancelledInput` |
| 入力デバイス | `.systemDefault` |
| 出力デバイス | `.systemDefault` |
| 送信 Voice Isolation Effect | `true` |
| Duck Other Audio | `true` |
| RTC 有効 route | `[.multipeer, .webRTC]` |
| VAD 感度 | `.standard` |
| 送信コーデック | `.mpeg4AACELDv2` |
| AAC-ELD v2 bitrate | `32_000` |
| Opus bitrate | `32_000` |
| マスター出力音量 | `1.0` |
| マスター出力ミュート | `false` |
| マスター出力 Voice Isolation Effect | `false` |
| 参加者別出力音量 | 未設定扱い |
| 参加者別 Voice Isolation Effect | 未設定扱い |

`resetAllSettings()` は local member、グループ一覧、参加者構成、group secret、接続中の group 選択を変更しない。

## 画面配置の対応

| 画面 | セクション | App 設定 |
|---|---|---|
| Call | Status Header | ローカルマイクミュート、マスター出力音量、マスター出力ミュート、マスター出力 Voice Isolation Effect |
| Call | Participants | 参加者別出力音量、参加者別 Voice Isolation Effect |

| Settings | Audio I/O | 入力デバイス、出力デバイス、送信 Voice Isolation Effect |
| Settings | Voice Activity | VAD 感度 |
| Settings | Transmit Codec | 送信コーデック、AAC-ELD v2 bitrate、Opus bitrate |

| Settings | Reset | 設定リセット |
