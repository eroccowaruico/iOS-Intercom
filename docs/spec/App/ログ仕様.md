# RideIntercom App ログ仕様

## 目的

本書は、作り直す RideIntercom App が `docs/requirements/log.md` をどう満たすかを定義する。

ログ基盤、レベル、禁止情報、共通 metadata は `docs/requirements/log.md` を正とする。本書では App が package の report / event をどの event 名へ変換するかだけを扱う。

## Logger

| 領域 | Logger label | 主な入力 |
|---|---|---|
| App lifecycle | `com.yowamushi-inc.RideIntercom.app` | App 起動、scene、deep link |
| RTC | `com.yowamushi-inc.RideIntercom.rtc` | `RTC.CallSessionEvent`、route metrics |
| Audio | `com.yowamushi-inc.RideIntercom.audio` | `SessionManager` report/event、Codec fallback、Mixer 起動 |
| Security | `com.yowamushi-inc.RideIntercom.security` | Keychain、credential 生成、invite validation |
| Settings | `com.yowamushi-inc.RideIntercom.settings` | ユーザー設定変更、reset |

## 共通方針

| 項目 | 仕様 |
|---|---|
| 初期化 | App 起動時に `LoggingSystem.bootstrap` を一度だけ実行する |
| 本文 | 可変値を埋め込まず、安定した event 名を使う |
| metadata | 調査に必要な値だけを入れる |
| operationID | 接続開始、media 開始、Audio Check、invite 受理など複数ログにまたがる操作で発行する |
| 高頻度処理 | audio frame、packet、tick ごとの通常ログは出さない |
| 失敗時 | package の `failed` report は warning / error へ変換し、`ignored` は原則 debug または notice にする |

## 記録禁止

| 情報 | 方針 |
|---|---|
| group secret | 記録禁止 |
| invite token | 記録禁止 |
| 認証 MAC / 鍵材料 | 記録禁止 |
| 音声 sample / encoded payload | 記録禁止 |
| 復号済み application data payload | 原則禁止 |
| peer / group の生 ID | 原則禁止。ハッシュまたは短縮値を使う |

## Diagnostics との関係

| 項目 | 方針 |
|---|---|
| Diagnostics | 直近の package report / event と metrics を画面表示する |
| Logs | 操作境界、失敗、fallback、handover だけを記録する |
| 高頻度値 | Diagnostics では表示してよいが、通常ログへは出さない |
| 機密値 | Diagnostics と Logs のどちらにも出さない |
