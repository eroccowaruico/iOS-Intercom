# RideIntercom UI 全体構成

## 目的

本書は、作り直す RideIntercom App の画面全体構成を定義する。

画面内の個別項目は `docs/spec/App/UI/画面項目定義.md` を入口とし、画面ごとの正本は `docs/spec/App/UI/Call.md`、`docs/spec/App/UI/Diagnostics.md`、`docs/spec/App/UI/Settings.md` とする。

状態遷移は `docs/spec/App/UI/画面・状態遷移.md` を正とする。

## 画面構成

| 階層 | 画面 | 目的 | 主な package 連携 |
|---|---|---|---|
| Root | `TabView` | 通話、診断、設定を切り替える | なし |
| Call | Groups | 接続対象 group を選ぶ | RTC credential 生成前の App 状態 |
| Call | Call | 接続、招待、ミュート、参加者音量を扱う | RTC / SessionManager / Mixer |
| Diagnostics | Diagnostics | package runtime report、metrics、fallback を確認する | RTC / SessionManager / Codec / Mixer |
| Settings | Settings | 入出力、codec、VAD、Audio Check、reset を扱う | SessionManager / Codec / VADGate / Mixer |

## タブ

| タブ | 表示名 | 役割 | 通常利用での位置づけ |
|---|---|---|---|
| Call | `Call` | 通話操作と参加者確認 | 主画面 |
| Diagnostics | `Diagnostics` | 状態確認と切り分け | ログを読まず現在状態を一覧で確認する画面 |
| Settings | `Settings` | 利用前調整と自己診断 | 通話前後の設定画面 |

## App が UI に出す情報

| 情報 | 出す画面 | 方針 |
|---|---|---|
| 接続状態 | Call / Diagnostics | RTC の state を App 表示へ要約する |
| 音声入出力 | Call / Settings / Diagnostics | SessionManager の snapshot / report を要約する |
| codec | Settings / Diagnostics | Settings はユーザー希望だけを扱い、実 codec / fallback は Diagnostics で確認する |
| route metrics | Diagnostics | RTC `RouteMetrics` を一覧カードへ要約する |
| ログ | 画面には直接出さない | `docs/spec/App/ログ仕様.md` に従い構造化ログへ出す |

## App が UI に出さない情報

| 情報 | 理由 |
|---|---|
| group secret / invite token | 機密情報 |
| 音声 sample / encoded payload | 機密かつ高頻度 |
| native WebRTC 型 | RTC package が隠す責務 |
| Audio Unit parameter の全量 | package 仕様を正とし、App 画面は必要な設定だけを持つ |
