# RideIntercom 画面項目定義

## 目的

本書は作り直し後の RideIntercom App の UI 項目定義の入口である。

画面ごとの項目はファイルを分け、1 ファイルが肥大化しないようにする。各画面の正本は下表の文書とする。

## 画面別定義

| 対象 | 正本 | 役割 |
|---|---|---|
| 共通構成 | `docs/spec/App/UI/共通項目.md` | `TabView`、共通ナビゲーション、共通表示方針 |
| Call | `docs/spec/App/UI/Call.md` | グループ選択、通話、参加者、入出力操作 |
| Diagnostics | `docs/spec/App/UI/Diagnostics.md` | Live pipeline と現在状態の一覧カード |
| Settings | `docs/spec/App/UI/Settings.md` | Audio Session、Audio I/O、Audio Check、codec、VAD、reset |
| アイコン | `docs/spec/App/UI/アイコン適用.md` | design system の SF Symbol 適用先 |

## 記載方針

| 項目 | 方針 |
|---|---|
| 対象画面 | `Call`、`Diagnostics`、`Settings` |
| 表先頭列 | UI項目を定義する表は、先頭列を `項目名（日本語）` に統一する |
| 親子関係 | 画面、セクション、子項目を分けて所属関係を追えるようにする |
| データ仕様 | 値の型、値域、参照元、表示用整形方法を記載する |
| 表示仕様 | 見え方、レイアウト、色、操作方法、補助表示の意味を記載する |
| 状態 | 接続状態、音声状態、診断状態など、画面から読み取るべき状態変化を記載する |
| 異常系 | 継続可能な `ignored` / fallback と、操作失敗の `failed` を分ける |
