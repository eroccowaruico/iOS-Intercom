# RideIntercom UI 共通項目

## 目的

本書は RideIntercom App の全画面で共通する UI 項目と表示方針を定義する。

## 共通定義

| 項目 | 内容 |
|---|---|
| ルート | `TabView` |
| 各タブ | `NavigationStack` |
| ローカルメンバー | `selectedGroup.members.first` を自分として扱う |
| リモートメンバー | `selectedGroup.members.dropFirst()` を相手一覧として扱う |
| メーター | `VoiceMeterView` は値文字列を直接出さず、レベルとピークを視覚表現する |
| URL 受理 | `rideintercom://join?token=...` を `onOpenURL` で受理し、成功時は Call タブへ戻す |

## 共通タブ

| 項目名（日本語） | 項目ID | ラベル | 目的 | データ仕様 | 表示仕様 |
|---|---|---|---|---|---|
| 共通タブ | なし | なし | 通話、診断、設定を役割ごとに分離する | `AppTab` | 下部タブバーとして常時表示 |
| 通話タブ | `callTab` | `Call` | 日常利用時の主画面へ移動する | 固定文字列 | `waveform.circle.fill` |
| 診断タブ | `diagnosticsTab` | `Diagnostics` | 音声・通信・認証の状態確認へ移動する | 固定文字列 | `gauge` |
| 設定タブ | `settingsTab` | `Settings` | 利用前調整と自己診断へ移動する | 固定文字列 | `gearshape.fill` |

## 共通表示方針

| 項目 | 方針 |
|---|---|
| 接続と media | control plane と media plane を混同させない |
| 設定値と実効値 | requested と selected / applied を分ける |
| 環境差異 | package report の `ignored` は非対応 no-op として表示し、致命扱いしない |
| 失敗 | `failed` は Call のエラー文言、Diagnostics の severity、ログに反映する |
| 高頻度値 | 画面は要約値だけを表示する。ログ確認を前提にしない |
| 機密値 | group secret、invite token、鍵材料、音声 payload は表示しない |

## 表示解釈

| 表示 | 解釈 |
|---|---|
| Input メーター | ローカルは送信前入力、参加者カードは受信後デコード済み入力を示す |
| Output メーター | 最終出力へ回した音声を示す |
| Participants 状態 | 接続、認証、受信、再生を別軸で読む |
| Diagnostics Overview | 1 カード 1 層で、ぱっと見て現在起きていることを確認する |
