# RideIntercom アイコン定義（SF Symbols）

本ドキュメントは RideIntercom のアイコン運用を定義するデザインシステム正本である。  
ここで定義するのは「意味」「ファミリー選定」「サイズ」「色」「レンダリング規約」であり、特定画面や実装条件ではない。  
画面ごとの具体的な採用シンボル、状態分岐、表示条件は `docs/spec/画面項目定義.md` を正本とする。

## 目的

- 必須: 同じ意味には同じファミリーを使う。
- 必須: 同じ場所には同じ種類のアイコンだけを出す。
- 必須: 色は装飾ではなく意味として運用する。
- 必須: サイズは利用用途ごとのトークンで統一する。
- 必須: SF Symbols の rendering mode、fill variant、variable value、multicolor を意味に従って使い分ける。
- 禁止: 実装都合だけでシンボルを選ぶこと。
- 禁止: 一時的な画面要件を本ドキュメントへ持ち込むこと。

## 適用範囲

- 対象: iOS / macOS の主要導線画面、診断導線画面、設定導線画面
- 対象要素: タブ、ツールバー、リスト行、カード見出し、状態表示、操作ボタン、設定項目、診断項目
- 対象アセット: SF Symbols を用いるすべてのアイコン

## 基本原則

### 意味優先

- 必須: アイコンは「見た目」ではなく「意味」で選ぶ。
- 必須: ひとつの意味領域に対して、原則ひとつのファミリーを割り当てる。
- 禁止: 同じ意味領域で別ファミリーを混在させること。

### スロット優先

- 必須: 同一 UI スロットでは同一の意味階層を維持する。
- 例: 行頭アイコンは「対象識別」、行末アイコンは「遷移示唆」、状態バッジは「状態伝達」に限定する。
- 禁止: 行頭に状態アイコンと対象識別アイコンを混在させること。
- 禁止: 行末に状態アイコンと操作アイコンを混在させること。

### 状態は同一ファミリー内で変化させる

- 必須: 通常、待機、成功、失敗、遮断などの状態差は、まず同一ファミリー内の variant と色で表現する。
- 必須: family をまたぐ変更は、意味カテゴリ自体が変わる場合に限る。
- 禁止: 同じ状態系で、通常時は `wifi`、異常時は `speaker` のように意味カテゴリを跨ぐこと。

### 接続成否と通信方式を分離する

- 必須: 「接続できているか」は Status として扱う。
- 必須: 「何の通信方式か」は Communication または Entity として扱う。
- 必須: 接続成否と通信方式は、同一アイコンで兼務させない。
- 必須: 接続成否は `wifi` 系で統一する。
- 必須: 通信方式や経路差分は `radiowaves` または `network` 系で扱う。
- 禁止: `radiowaves` を接続済みの最終結果アイコンとして使うこと。
- 禁止: `wifi` を Multipeer / WebRTC の方式差分アイコンとして使うこと。

## アイコン意味領域

RideIntercom ではアイコンを次の意味領域に分ける。

| 意味領域 | 説明 |
|---|---|
| Navigation | 画面・導線・セクションへの移動や所属を示す |
| Entity | グループ、参加者、端末、経路など対象そのものを識別する |
| Status | 接続可否、認証可否、稼働可否など状態を示す |
| Communication | 通信の存在、送受信、伝送チャネル、経路レイヤーを示す |
| Input | マイク、入力取得、入力可否、入力レベルを示す |
| Output | スピーカー、再生可否、出力レベルを示す |
| Audio Processing | 音声検出、波形、音声処理、エフェクト、ダッキングなど音声処理系を示す |
| Action | 作成、共有、削除、停止など操作を示す |
| Diagnostics | 計測、検査、要約、観測を示す |

## シンボルファミリー規約

### Navigation

| ファミリー | 代表シンボル | 意味 |
|---|---|---|
| section / feature | `waveform.circle.fill`, `gauge`, `gearshape.fill` | 機能入口、導線、主要カテゴリ |
| directional | `chevron.right` | 遷移、展開、次階層 |

- 必須: Navigation は feature 系または directional 系のみを使う。
- 禁止: Navigation に状態系アイコンを使うこと。

### Entity

| ファミリー | 代表シンボル | 意味 |
|---|---|---|
| people | `person.2`, `person.3`, `person.text.rectangle` | グループ、参加者、所属、ローカル/リモート主体 |
| topology | `network` | 構成、ネットワーク経路、関係性 |

- 必須: Entity は「誰・何」を示す。
- 禁止: Entity アイコンに接続成功/失敗の意味を持たせること。

### Status

| ファミリー | 代表シンボル | 意味 |
|---|---|---|
| wifi | `wifi`, `wifi.slash`, `wifi.exclamationmark` | 接続状態の良否、可用性、接続試行 |
| seal / lock | `checkmark.seal`, `xmark.seal`, `lock.open` | 認証、許可、保証、未確定 |
| alert | `exclamationmark.triangle.fill` | 異常、要注意、回復不能 |

- 必須: 接続状態の最終結果は `wifi` 系で統一する。
- 必須: 認証状態は `seal / lock` 系で統一する。
- 禁止: `antenna` や `speaker` を接続良否の最終状態に使うこと。

### Communication

| ファミリー | 代表シンボル | 意味 |
|---|---|---|
| radiowaves | `dot.radiowaves.left.and.right`, `antenna.radiowaves.left.and.right` | 通信レイヤー、送受信チャネル、電波的存在 |
| network | `network` | 通信方式、経路構成、トポロジー差分 |
| paperplane | `paperplane`, `paperplane.fill` | 送信動作、送信キュー、送信実行 |

- 必須: Communication は「通信の存在」や「送受信動作」を表す。
- 必須: 通信方式差分は `network` または `radiowaves` で表す。
- 禁止: Communication を接続成否の最終状態表示へ転用すること。

### Input

| ファミリー | 代表シンボル | 意味 |
|---|---|---|
| microphone | `mic`, `mic.fill`, `mic.slash.fill` | マイク、入力可否、送話可否 |
| input meter | `microphone.and.signal.meter` | 入力レベル、入力状態の可視化 |

- 必須: Input は入力デバイスまたは入力状態のみを表す。
- 禁止: Input ファミリーを受信再生や出力制御へ転用すること。

### Output

| ファミリー | 代表シンボル | 意味 |
|---|---|---|
| speaker | `speaker.wave.2.fill`, `speaker.slash.fill`, `speaker.square.fill` | 再生、出力可否、外部音量との関係 |

- 必須: Output は再生側の意味に限定する。
- 禁止: Output ファミリーを接続状態や入力状態へ転用すること。

### Audio Processing

| ファミリー | 代表シンボル | 意味 |
|---|---|---|
| waveform | `waveform`, `waveform.slash`, `waveform.badge.mic` | 波形、音声検出、音声処理、音声処理の有無 |
| compute | `cpu`, `cpu.fill` | エンコード、処理段、演算段 |

- 必須: Audio Processing は音そのもの、または音処理段を示す。
- 禁止: `slider.horizontal.3` を音声処理状態そのものに使うこと。

### Action

| ファミリー | 代表シンボル | 意味 |
|---|---|---|
| creation / share / destroy | `plus`, `square.and.arrow.up`, `trash`, `xmark.circle.fill` | 明示操作 |

- 必須: Action は押下可能な操作に限定する。
- 禁止: Action アイコンを状態表示専用スロットへ置くこと。

### Diagnostics

| ファミリー | 代表シンボル | 意味 |
|---|---|---|
| observation | `gauge`, `checklist`, `clock.arrow.circlepath` | 観測、確認、要約、診断 |

- 必須: Diagnostics は計測や観測の文脈に限定する。
- 禁止: 主要導線の主操作へ診断系アイコンを持ち込むこと。

## 排他規則

### 接続状態スロット

- 必須: 接続状態スロットは `wifi` 系または `alert` 系のみを使う。
- 許可条件付き: 通信レイヤーがあるが主状態ではないことを示す中間表現に限り `radiowaves` 系を使える。
- 必須: 中間表現を追加する場合でも、接続の最終状態アイコンそのものは `wifi` 系から変えない。
- 禁止: `speaker`、`mic`、`person` を接続状態スロットへ使うこと。

### 音声入出力スロット

- 必須: 入力スロットは `microphone` 系のみを使う。
- 必須: 出力スロットは `speaker` 系のみを使う。
- 禁止: 入力スロットに `waveform` だけを置くこと。
- 禁止: 出力スロットに `wifi` や `radiowaves` を置くこと。

### 音声処理スロット

- 必須: VAD、波形、FX、ダッキング、音声処理は `waveform` 系または `compute` 系で扱う。
- 禁止: 音声処理の ON/OFF を `mic` や `speaker` の mute 状態で代用すること。

### 対象識別スロット

- 必須: グループや参加者などの対象識別には `people` 系または `topology` 系を使う。
- 禁止: 対象識別に状態アイコンを直接使うこと。

### 通信方式スロット

- 必須: Multipeer / WebRTC / Internet / Local などの方式差分は、接続状態スロットではなく通信方式スロットで表す。
- 必須: 通信方式スロットは `radiowaves` または `network` 系のみを使う。
- 必須: 通信方式スロットの色は、原則 `textSecondary` を基準とし、成功/失敗色で接続成否を表さない。
- 禁止: 通信方式スロットに `wifi` 系を使うこと。
- 禁止: 通信方式スロットで接続中/接続済み/失敗を兼務させること。

## サイズ規約

サイズは `docs/design system/デザインシステム.md` のサイズトークンを正本とし、意味用途ごとに次を適用する。

| 用途 | サイズトークン | 説明 |
|---|---|---|
| タブ・主要機能入口 | システム標準 | タブバーや主要ナビゲーションの標準サイズを使う |
| 行頭の主対象アイコン | `sizeIconM` | グループ、参加者、主要状態ブロックの主対象 |
| 行頭の補助対象アイコン | `sizeIconS` | 診断行、設定行、補助情報行 |
| 主状態アイコン | `sizeIconM` | 接続、認証、入出力、通話状態の主表示 |
| 補助状態アイコン | `sizeIconS` | サブ状態、補助説明、方式差分、二次情報 |
| パイプライン段アイコン | `sizeIconL` | エンコードや処理フローなど段構造 |
| アイコン単体ボタン | `sizeTapPrimary` | ミュート、切断など押下領域を持つ操作 |

### サイズ運用原則

- 必須: サイズは「行」ではなく「意味役割」で決める。
- 必須: 同じ行の中でも、主対象と補助対象、主状態と補助状態、表示と操作でサイズは変わってよい。
- 必須: 親子関係、情報の主従、左右の意味差がある場合は、それに応じてサイズ差を許可する。
- 必須: サイズ差は意味差を強めるためにのみ使う。
- 禁止: 実装都合や見た目調整だけでサイズ差を付けること。
- 禁止: 同じ意味役割なのに画面ごとでサイズを変えること。
- 禁止: 同一リスト内で意味差がないのにサイズ差を付けること。

### 主従関係の扱い

| 関係 | 推奨サイズ関係 | 例 |
|---|---|---|
| 主対象と補助対象 | 主対象 > 補助対象 | グループ本体アイコンと補足経路アイコン |
| 主状態と補助状態 | 主状態 > 補助状態 | 接続成否と通信方式差分 |
| 情報表示と操作 | 情報表示 < 操作 | 情報行の補助アイコンと押下可能な単体ボタン |
| 段構造と行内情報 | 段構造 > 行内情報 | パイプライン段アイコンと通常行アイコン |

### 同一行内でサイズ差を認める条件

- 必須: 左右で意味が異なる場合、サイズ差を認める。
  例: 左は主対象、右は補助状態
- 必須: 親子関係がある場合、親を大きく子を小さくしてよい。
  例: 接続状態の主表示と方式差分の補助表示
- 必須: 単体押下ボタンは、同じ行内の情報アイコンより大きくてよい。
- 禁止: 左右に散らしているだけで、意味差がないのにサイズ差を付けること。

## 色規約

色は `docs/design system/デザインシステム.md` のカラートークンを正本とし、意味は下表で統一する。

| 色トークン | 意味 |
|---|---|
| `textPrimary` | 主要操作や主要情報に対する標準表示 |
| `textSecondary` | 補助識別、補助情報、非主要状態 |
| `textTertiary` | 遷移示唆、弱い補助、背景的存在 |
| `neutral` | 非稼働、未接続、待機前の基準状態 |
| `info` | 情報提示、観測中、受信系の情報状態 |
| `success` | 成功、接続完了、再生中、通過中、正常稼働 |
| `warning` | 接続中、保留、注意、遷移中、中間状態 |
| `danger` | ミュート、遮断、削除、異常、失敗 |
| `accentColor` | 主要な明示操作 |

- 必須: 色だけで意味を成立させない。
- 必須: 色が変わる場合でも、同一ファミリーまたは variant の中で状態差を表す。
- 禁止: 同じ意味に画面ごとで別色を当てること。

## SF Symbols レンダリング規約

### Rendering Mode

- 必須: 原則は `.hierarchical` または単色運用を基準とする。
- 必須: 状態意味を色で伝えるアイコンは単一意味色で描画する。
- 許可条件付き: オブジェクト構成を自然物として見せる必要がある場合のみ `.palette` を使える。
- 許可条件付き: Apple 標準の意味を強く持つシンボルに限り `.multicolor` を使える。
- 禁止: 状態表示アイコンへ装飾目的で `.multicolor` を使うこと。

### Fill Variant

- 必須: `fill` は「能動」「確定」「強調」に使う。
- 必須: non-fill は「待機」「中立」「補助」に使う。
- 必須: 同一カテゴリで fill / non-fill を状態差として使い分ける。
- 禁止: 同階層で意味差なく fill / non-fill を混在させること。

### Variable Value

- 必須: variable value 対応シンボルは、段階的な量や進行度を表す時のみ使う。
- 必須: variable value を使う場合は実値に連動させる。
- 禁止: 可変対応シンボルを常に `1.0` 固定で使い、単なる別アイコンのように扱うこと。

### Multicolor

- 必須: multicolor はシンボル固有の意味が UX 上有益な場合のみ使う。
- 禁止: ブランド感や賑やかしのためだけに multicolor を使うこと。
- 禁止: 同一状態群で一部だけ multicolor にすること。

## 運用規約

- 必須: 新しい意味領域を追加する場合は、先に本ドキュメントへ意味領域・ファミリー・色・サイズを定義する。
- 必須: 画面固有の採用シンボルは `画面項目定義.md` に記載する。
- 必須: 実装が本ドキュメントと衝突する場合は、実装ではなく本ドキュメントの意味体系を優先して是正する。

## アンチパターン

- 禁止: 実装で今たまたま使っているシンボルを、そのままデザインシステム正本へ昇格させること。
- 禁止: 同じ「接続状態」に `wifi`、`antenna`、`speaker` を混在させること。
- 禁止: 行頭アイコンに対象識別と状態表示を同時に背負わせること。
- 禁止: `slider` 系を音声処理状態そのものの意味で使うこと。
- 禁止: `mic` 系で出力や受信の状態を表すこと。
- 禁止: `speaker` 系で接続状態や認証状態を表すこと。

## 実装適用ガイド

本節は、現在の企画・アーキテクチャ・実装要件に対して、どの意味領域とファミリーを採用すべきかを固定する補助定義である。  
ここでは「どの画面のどの場所に何を出すか」ではなく、「その場所で使うべき意味カテゴリ」を定義する。  
個別画面での最終採用シンボルは `画面項目定義.md` で決める。

### 最優先の固定ルール

| 表したいもの | 使うべき意味領域 | 使うべきファミリー | 使ってはいけないもの |
|---|---|---|---|
| 接続できているか | Status | wifi / alert | radiowaves, network, speaker |
| 何の方式で通信しているか | Communication | radiowaves / network | wifi |
| 認証されているか | Status | seal / lock / alert | wifi, radiowaves |
| 送信しているか | Communication | paperplane または antenna/radiowaves | wifi |
| マイクが使えるか | Input | microphone | waveform, speaker |
| スピーカーから再生されるか | Output | speaker | microphone, wifi |
| VAD / FX / ダッキング / 音声処理 | Audio Processing | waveform / compute | microphone, speaker, wifi |

- 必須: 接続状態と通信方式は別スロットに分ける。
- 必須: 同じ行やカード内に両方を出す場合でも、片方は成否、片方は方式だけを表す。
- 禁止: `wifi` と `radiowaves` を同じ「接続状態」の意味で併用すること。

### 現行導線に対する適用

| 対象 | 表したい意味 | 採用すべき意味領域 | 採用すべきファミリー | 禁止ファミリー |
|---|---|---|---|---|
| タブバーの主要機能入口 | 機能カテゴリ | Navigation | section / feature | Status, Communication |
| ツールバーのグループ一覧復帰 | 導線・所属先 | Navigation | feature または people | Status |
| グループ一覧の行頭 | グループという対象識別 | Entity | people | Status, Communication |
| グループ一覧の行末 | 遷移示唆 | Navigation | directional | Action, Status |
| 通話ヘッダーの接続状態 | 接続良否の最終状態 | Status | wifi / alert | speaker, microphone, people |
| 通話ヘッダーの通信方式表示 | Multipeer / WebRTC / Local / Internet の方式差分 | Communication | radiowaves / network | wifi, speaker |
| 通話ヘッダーの補助通信状態 | 通信レイヤーの存在、中間状態 | Communication | radiowaves | wifi 完了状態の代用としての他ファミリー |
| 通話ヘッダーの認証状態 | 認証済み、保留、失敗 | Status | seal / lock / alert | wifi, speaker |
| Connect / Disconnect など通信開始停止の主操作 | 明示操作 | Action | creation / destroy または Communication の送信動作補助 | Status を主操作アイコンとして使うこと |
| 参加者行の行頭 | 参加者という対象識別 | Entity | people | Status |
| 参加者ごとの受信音量・再生可否 | 出力状態 | Output | speaker | microphone, wifi |
| マスター出力のミュート・有効 | 出力可否 | Output | speaker | Status, Input |
| ローカルマイクのミュート・有効 | 入力可否 | Input | microphone | Output, Status |
| 入力レベル表示 | 入力量の可視化 | Input | input meter | waveform 単独, speaker |
| 受信音声処理、VAD、FX、ダッキング | 音声処理状態 | Audio Processing | waveform | microphone, speaker, slider |
| エンコード・送信段 | 処理段、送信段 | Audio Processing / Communication | compute, paperplane | wifi, speaker |
| 診断サマリ行の先頭 | 観測・要約 | Diagnostics | observation | Action, Status |
| 設定セクション見出し | 設定カテゴリ | Navigation または Entity/IO の抽象識別 | feature, speaker, microphone, waveform のいずれかを意味に応じて限定採用 | 状態系を見出しへ使うこと |

### 予約定義

今後の実装で追加しやすいよう、意味カテゴリを先に予約する。

| 予約対象 | 予定される意味 | 採用すべき意味領域 | 採用すべきファミリー | 補足 |
|---|---|---|---|---|
| ローカル待受中の通信表示 | 接続完了ではない通信待機 | Communication | radiowaves | `wifi` 完了状態と混同しない |
| Media Plane 未起動の control-only 状態 | 接続済みだが通話未開始 | Communication または Status の補助表現 | radiowaves を優先 | 接続成功そのものは `wifi`、補助状態だけを追加で示す |
| Multipeer と WebRTC の差 | 通信方式差分 | Communication | network または radiowaves | 接続成否は `wifi` で別管理する |
| インターネット経由とローカル経由の差 | 経路構成 | Communication | network, radiowaves | 成否表現と混ぜない |
| 音声ダッキング設定 | 他音声への干渉制御 | Audio Processing または Output 補助 | waveform または speaker 補助識別 | 接続状態やミュート状態とは分離する |
| Sound Isolation など入力処理 | 入力処理 | Audio Processing | waveform | mic ミュートと混同しない |
| 送信キュー、送信保留 | 送信進行度 | Communication | paperplane | 接続状態は表さない |
| 受信監視、受信サマリ | 観測・受信状況 | Diagnostics または Communication | observation, radiowaves | 出力可否とは分離する |
| グループ切替候補や優先対象 | 対象の強調 | Entity | people | 状態色は許可、状態アイコン化はしない |

### 実装時の選定手順

1. まず、そのアイコンが「対象」「状態」「通信」「入出力」「音声処理」「操作」「診断」のどれかを決める。
   接続成否なら `状態`、Multipeer / WebRTC / Local / Internet なら `通信` と判断する。
2. 次に、その UI スロットが「行頭識別」「行末遷移」「状態表示」「操作ボタン」のどれかを決める。
3. 上の 2 つが決まったら、本ドキュメントの意味領域と排他規則からファミリーを選ぶ。
4. 色は状態意味でのみ変え、ファミリーは意味が変わらない限り変えない。
5. 具体シンボル名と状態分岐は `画面項目定義.md` に落とす。
