# RideIntercom デザインシステム

本ドキュメントは、現行画面で使用している見た目・レイアウト・コントロール仕様を共通化し、実装を強制可能にするための正本である。
本ドキュメントに違反するUI変更は、実装済みであっても不合格とする。

## 強制度

- 必須: 実装時に必ず満たす。未達はマージ不可。
- 禁止: 実装してはならない。検出時は即修正。
- 許可条件付き: 明示した条件に一致する場合のみ許可。

## 適用範囲

- 対象画面: 主要導線画面群、診断導線画面群、設定導線画面群。
- 対象要素: 背景、カード、項目色、角丸、余白、文字、ボタン、タブ、Form Section、メーター、状態表示、アクセシビリティ。
- 対象プラットフォーム: iOS、macOS。
- 基準プラットフォーム: iOS。
- macOS適用方針: iOS基準の構造、状態、アクセシビリティ、意味を維持したうえで、macOS固有の入力・ウィンドウ特性に追従する。

## iOS基準定義

- 必須: 本ドキュメント内の値、トークン、レイアウト規約、文字規約は iOS 実装を基準値とする。
- 必須: 新規UI仕様は iOS で成立する具体値を先に定義し、その後に macOS へ同等意味で展開する。
- 必須: macOS 対応は iOS 基準を崩さない範囲で行い、意味差分を生む見た目差分を追加しない。
- 禁止: macOS 側の都合を理由に iOS 基準のトークン値や状態表現を変更すること。

## 実装原則

- 必須: iOS 実装を正とし、macOS は同一の画面構造、同一の状態管理、同一のアクセシビリティIDで追従する。
- 必須: OS差分は Adapter 層で吸収し、View へOS条件分岐を入れない。
- 必須: 主要導線画面は中核体験に必要な情報と操作のみを表示し、診断情報は診断導線画面へ集約する。
- 必須: 設定導線画面は Form と Section の標準構成を維持し、入出力設定、送信方式設定、自己診断設定を分離する。
- 必須: 状態伝達は色のみで完結させない。テキスト、アイコン、accessibility label/value を併用する。

## SwiftUI優先原則

- 必須: UI 実装は SwiftUI 標準コンポーネントを第一選択とする。
- 必須: `TabView`、`NavigationStack`、`List`、`Form`、`Section`、`Button`、`Toggle`、`Slider`、`Picker`、`ShareLink`、`Label` など、意味が一致する標準要素を優先する。
- 必須: カスタム UI は、標準コンポーネントでは意味を満たせない場合に限り導入する。
- 必須: カスタム UI を導入する場合は、標準コンポーネントでは満たせない理由を仕様で説明できなければならない。
- 禁止: 見た目を少し変えたいだけで独自コンポーネントを作ること。
- 禁止: 標準コンポーネントが持つアクセシビリティ、フォーカス、プラットフォーム適応を捨てる実装。

## 意味モデル

本デザインシステムは、見た目ではなく意味を統一対象とする。  
同じ意味は、同じ構造、同じ状態、同じトークン、同じ操作規約で表現する。

| 意味レイヤー | 説明 |
|---|---|
| Screen | 画面全体の役割。主要導線、診断導線、設定導線など |
| Section | 情報群または操作群のまとまり |
| Item | 行、カード、メーター、ボタンなどの最小意味単位 |
| State | その Item が今どういう状況か |
| Action | ユーザーが明示的に発火する操作 |

- 必須: UI 変更はまず「何の意味レイヤーが変わるのか」を説明できなければならない。
- 必須: Item の状態と Action を混同しない。
- 禁止: 同じ見た目で意味が異なるものを同一コンポーネントとして扱うこと。

## UI Stack構造原則

- 必須: 画面は `Screen > Section > Item` の順で意味を積み上げる。
- 必須: 上位ほど役割を、下位ほど具体状態を持つ。
- 必須: 同一 Section 内の Item は、比較可能な情報か、同一目的の操作に限定する。
- 必須: 主要導線画面は「状態把握 > 主要操作 > 対象一覧 > 補助情報」の順で積む。
- 必須: 診断導線画面は「要約 > 切り分け > 詳細」の順で積む。
- 必須: 設定導線画面は「環境依存設定 > 通話方式設定 > 自己診断設定」の順で積む。
- 禁止: 主要導線と診断導線の責務を同一 Section に混在させること。

## 状態モデル

UI における状態は、以下の意味として統一する。

| 状態種別 | 定義 | 代表的な扱い |
|---|---|---|
| Ideal State | もっとも期待通りに動作している状態 | 主情報と主操作を素直に見せる |
| Loading State | データや遷移結果を待っている状態 | プレースホルダ、進行中文言、操作制限 |
| Partial State | 一部だけ利用可能、一部だけ未成立の状態 | 利用可能範囲を示し、全体失敗扱いしない |
| Error State | ユーザーが認識すべき失敗や異常がある状態 | 文言、色、アイコン、必要なら再試行導線 |
| Blank State | まだ生成されていない初期状態 | 次に取るべき作成行動を示す |
| Empty State | データは概念上存在するが、結果が 0 件の状態 | 空である理由と次の行動を示す |
| Disabled State | 条件未成立により操作不能な状態 | 押せないことと理由を明示する |

### Ideal State

- 必須: 主要導線画面は Ideal State を基準に構造を決める。
- 必須: Ideal State では余計な診断や例外表示を前面に出さない。

### Loading State

- 必須: Loading は「まだ決まっていない」の意味で使う。
- 必須: Loading 中は成功/失敗色を確定表示しない。
- 必須: Loading 中に押せない操作は disable するか、重複発火防止を入れる。
- 禁止: Loading を Error の代替に使うこと。

### Partial State

- 必須: Partial は「一部成立・一部未成立」を意味する。
- 必須: 接続済みだが音声未開始、入力は生きているが出力先が不確定、などは Partial として扱う。
- 必須: Partial は全体失敗として塗りつぶさない。
- 必須: Partial では使える機能を残し、失われている機能だけを補助表示する。

### Error State

- 必須: Error はユーザーが認知すべき失敗に限る。
- 必須: Error は danger 色、文言、必要なら再試行導線を持つ。
- 必須: Error でも既存情報の読解を阻害しない。
- 禁止: 回復可能な待機や一時的未成立を Error 扱いすること。

### Blank State

- 必須: Blank State は「まだ何も作っていない」を意味する。
- 必須: Blank State では原因説明より次の第一行動を優先する。
- 例: グループ未作成時の「Create a group to start a call.」

### Empty State

- 必須: Empty State は「概念はあるが対象件数が 0」を意味する。
- 必須: Empty State は Error にしない。
- 例: 参加者一覧が空、検索結果が 0 件

### Disabled State

- 必須: Disabled は「押してはいけない」ではなく「今は成立条件を満たしていない」を意味する。
- 必須: Disabled の理由を推測可能にする。
- 必須: Disabled は opacity 低下だけで済ませず、文言や状態との整合を取る。
- 禁止: 実際には押してよい操作を見た目だけ disable にすること。

## Enable / Disable 規約

- 必須: Enable は「今この操作が成立し、押下可能」であることを意味する。
- 必須: Disable は「押下不能であり、押下しても結果が成立しない」ことを意味する。
- 必須: 進行中の重複防止は disable を使ってよい。
- 必須: 権限未取得、依存データ未取得、競合中など、成立条件が欠ける場合のみ disable を使う。
- 禁止: 単に目立たせたくない操作を disable にすること。
- 禁止: 見た目を muted にしたいだけで disable を使うこと。

## macOS フォアグラウンド / バックグラウンド規約

- 必須: macOS では window focus と app active の差を前提に設計する。
- 必須: フォアグラウンド時は主要操作、主要状態、入力系の可読性を最優先する。
- 必須: バックグラウンド時は意味を維持しつつ、視覚強度だけを下げてよい。
- 必須: バックグラウンド時も状態の意味そのものを変えてはならない。
- 必須: Active/Inactive の差は色の彩度、material の強度、補助情報の強弱で表現し、構造は変えない。
- 禁止: バックグラウンド遷移で項目を消すこと。
- 禁止: バックグラウンド時に success を danger へ変えるなど、意味差を生むこと。
- 必須: macOS でのバックグラウンド時も、接続維持や通話継続などの状態そのものは正しく表示し続ける。

### 固定サイズが必要な項目

- 必須: 下表の項目のみ固定 pt 指定を許可する。
- 禁止: 下表以外で .font(.system(size: ...)) を使うこと。

| 項目 | 固定サイズ | 条件 | 理由 |
|---|---|---|---|
| パイプライン接続子 ">" | 11pt | 1文字記号を中央配置する場合のみ | 文字としての可読性とレイアウト安定性を維持するため |

### テキスト運用ルール

- 必須: タイトル系は textNavigationTitle または textSectionTitle 以外を使わない。
- 必須: ラベル系は textSubheadlineStrong、textCaptionStrong、textCaption のいずれかを用途で選ぶ。
- 必須: 補助情報は textSecondary または textTertiary を使う。
- 必須: 異常状態の文言は danger を使う。
- 必須: 診断値、メーター値、パーセンテージなど比較対象となる数値は monospacedDigit を使う。
- 必須: 長文になりうる項目は lineLimitFlexible を使い、固定1行にしない。
- 許可条件付き: 狭幅時に限り、scaleFactorTight を短文項目へ適用できる。
- 必須: 診断導線画面以外では、状態ラベルと操作ラベルに textCaption2 を使用しない。
- 必須: 診断導線画面以外では、状態ラベルと操作ラベルに textCaption を使う場合は補助情報に限定する。
- 禁止: 主要導線画面と設定導線画面の主要状態値や主要操作へ caption 系のみを使うこと。

## コントロール共通仕様

### 主要ボタン

- 必須: 実装は appProminentButtonStyle() を使用する。
- 必須: スタイルは .borderedProminent、背景は buttonProminentBackground、文字色は buttonProminentForeground を使用する。
- 必須: サイズは .controlSize(.large) とする。
- 必須: iOS の表示高さは sizeButtonPrimaryHeight 以上を維持する。
- 必須: macOS の表示高さは sizeButtonMacMinHeight 以上を維持する。

### 補助ボタン

- 必須: 補助操作は .buttonStyle(.bordered) を使用する。
- 必須: 状態反転型は danger と accentColor の2値で切り替える。
- 必須: iOS の表示高さは sizeButtonSecondaryHeight 以上を維持する。
- 必須: macOS の表示高さは sizeButtonMacMinHeight 以上を維持する。

### アイコン単体ボタン

- 必須: 押下領域は最小 44x44。
- 必須: accessibilityLabel を設定する。
- 必須: 状態を持つ場合は accessibilityValue を設定する。

### スライダー

- 必須: 音量系は既定では 0...1 の正規化範囲を使用する。通話画面のマスター出力スライダーのみ、1.0 を通常最大、1.0 超過を Boost とする 0...2.0 の範囲を使用できる。
- 必須: 数値表示を併記する。
- 必須: 閾値系は最小値、最大値、スナップ間隔を固定し表示する。

### トグル

- 必須: 環境依存機能は利用可能なときのみ表示する。

### リスト行とカード

- 必須: リスト行は行全体タップ可能。
- 必須: 削除操作は swipeActions と contextMenu の両方を提供する。
- 必須: カードは padding(12) と radiusCard を使用する。
- 必須: カード外側余白は最小で spaceM を確保し、隣接カード同士が視覚的に分離されること。
- 必須: カード境界は borderWidthCard と cardBorder を併用し、背景面と同化させない。
- 禁止: カード境界の省略や、背景面と識別できない低コントラスト設定。

### 空状態

- 必須: 主要導線画面内の各空状態は同一の空状態表現ルールを使用する。
- 必須: 空状態は空状態カードで表示し、cardMaterial、cardBorder、radiusCard、borderWidthCard、spaceXL を適用する。
- 必須: 空状態の本文は textSubheadline を使い、textSecondary で表示する。
- 必須: 空状態カードは画面幅いっぱいに貼り付けず、左右に spaceScreen 以上の余白を確保する。

## 画面コンテナ共通仕様

### アプリ全体

- 必須: ルート構成は TabView と NavigationStack を維持する。
- 必須: タブは主要導線、診断導線、設定導線の3分類固定。

### 主要導線画面

- 必須: 背景は callScreenBackground を使用する。
- 必須: 主要コンテナは ScrollView と VStack、標準縦間隔は spaceScreen。
- 必須: 通話ヘッダー、参加者カード、空状態カードは cardMaterial、cardBorder、radiusCard、borderWidthCard を使用する。
- 必須: 主要導線画面内の空状態表示は、同一ルールの空状態カードを使用する。

### 診断導線画面

- 必須: 診断行は diagnosticsCardMaterial と radiusCard でカード化する。
- 必須: 診断本文は用途で使い分ける。標準は textFootnoteMono（高密度一覧）、強調値のみ textBodyMono を使用する。
- 必須: パイプラインは5ステップ固定、等幅配置、接続子は sizeConnector。
- 必須: 診断導線画面は通常導線ではなく障害切り分け導線として扱い、一覧性を優先する。
- 必須: 診断導線画面の高密度表示でも Dynamic Type 追従を維持し、固定ptでの直接指定を行わない。
- 禁止: 主要導線画面と設定導線画面の本文へ診断導線向け高密度タイポグラフィを流用すること。

### 設定導線画面

- 必須: Form は .grouped を使用する。
- 必須: Section 順序は入出力設定、送信方式設定、自己診断設定。
- 必須: デバイス選択は .menu、少数択は .segmented を使用する。

### 背景とカードコントラスト

- 必須: 画面背景とカード面は常に識別可能な差を持たせる。
- 必須: カードは面と枠線の両方で区別し、cardBorder の不透明度は sizeCardMinContrastBorderOpacity 以上を維持する。
- 必須: 主要導線画面は callScreenBackground + cardMaterial、診断導線画面はシステム背景 + diagnosticsCardMaterial、設定導線画面は grouped Form 背景を維持する。
- 禁止: 背景とカード面が実質同色となる設定。

## 状態表示仕様

| 状態種別 | 色 | 補助表現 | 運用ルール |
|---|---|---|---|
| 接続成功 | success | テキスト（接続済み等）+ アイコン | 色単独で表現しない |
| 接続中/待機 | warning | テキスト（接続中/待機中等）+ アイコン | 色単独で表現しない |
| 未接続/非稼働 | neutral | テキスト（未接続/非稼働等）+ アイコン | 色単独で表現しない |
| 異常/失敗/ミュート | danger | テキスト（異常/失敗/ミュート等）+ アイコン | 色単独で表現しない |
| 情報提示/受信中 | info | テキスト + アイコン | 注意喚起色と混同しない |

- 必須: 状態行には色、アイコン、文言を同時に含める。
- 禁止: 状態意味を色だけで表現する実装。

## テキスト色規約

テキスト色は、内容の重要度と状態意味の 2 軸で決める。  
状態色を使うのは「そのテキスト自体が状態値である時」に限る。

| テキスト種別 | 既定色 | 状態連動時の許可色 | 禁止事項 |
|---|---|---|---|
| 画面タイトル | `textPrimary` | なし | 状態色へしない |
| セクション見出し | `textPrimary` | なし | 状態色へしない |
| 行タイトル / カード主タイトル | `textPrimary` | `danger` のみ許可条件付き | 通常状態で success / warning にしない |
| 補助見出し | `textSecondary` | `danger`, `success`, `warning`, `info` | 状態意味がないのに色を持たせない |
| 主状態値 | `textPrimary` または状態色 | `success`, `warning`, `danger`, `info`, `neutral` | 同一状態で画面ごとに色を変えない |
| 補助状態値 | `textSecondary` または状態色 | `success`, `warning`, `danger`, `info`, `neutral` | 状態値でない補助文へ状態色を使わない |
| 説明文 | `textSecondary` | なし | 状態色へしない |
| 最小補助文 | `textTertiary` または `textSecondary` | なし | 注意喚起を担わせない |
| エラー本文 | `danger` | なし | neutral や warning に落とさない |

- 必須: 状態値でない本文は `textPrimary` / `textSecondary` / `textTertiary` のいずれかで表す。
- 必須: 状態色を使う時は、そのテキストが状態判断の主体でなければならない。
- 禁止: 状態アイコンがあるからという理由だけで隣接テキストまで同色に染めること。

## コントロール色規約

コントロール色は「何の操作か」と「今その操作がどの状態か」で決める。  
情報表示の色規約をそのまま操作へ流用してはならない。

| コントロール種別 | 通常時 | 強調時 | 反転/破壊/停止時 | 無効時 |
|---|---|---|---|---|
| 主要ボタン | `buttonProminentBackground` + `buttonProminentForeground` | 同左 | `danger` を許可 | 彩度を落とすが意味色へしない |
| 補助ボタン | `textPrimary` / bordered | `accentColor` を許可 | `danger` を許可 | opacity 低下 + 押下不可 |
| アイコン単体ボタン | `buttonProminentBackground` または bordered | `accentColor` を許可 | `danger` を許可 | opacity 低下 + 押下不可 |
| Toggle | システム標準 | システム標準 | 使用しない | 押下不可で示す |
| Slider | システム標準 | なし | 使用しない | 押下不可で示す |
| Picker | システム標準 | なし | 使用しない | 押下不可で示す |

- 必須: 破壊、停止、遮断、解除系の操作だけが `danger` を使える。
- 必須: `Connect` のような開始操作は success ではなく、主要操作色または標準操作色を使う。
- 必須: `Disconnect`、`Delete`、`Mute`、`Stop` のような停止・遮断操作は `danger` を使ってよい。
- 禁止: 現在状態が success だからという理由で、その状態を変える操作ボタンも success 色にすること。
- 禁止: 情報表示の success / warning / info を、そのまま押下操作色へ流用すること。

## 状態遷移時の色適用規約

状態遷移に伴い色が変わるのは、状態を表す要素だけである。  
レイアウト内の全要素を同時に染めてはならない。

| 要素 | 状態遷移で色が変わってよいか | 規則 |
|---|---|---|
| 接続状態アイコン | 変わってよい | `neutral -> warning -> success -> danger` |
| 接続状態テキスト | 変わってよい | 主状態値として扱う時のみ |
| 経路ラベル | 原則変えない | `textSecondary` 基準。方式差分は意味で表し、成否色を持たせない |
| セクション見出し | 変えない | 状態色にしない |
| 説明文 | 変えない | エラー本文を除く |
| 主要ボタン | 状態に応じて変えてよい | ただし操作意味が変わる時だけ |
| アイコン単体ボタン | 状態に応じて変えてよい | mute / unmute などの反転操作に限る |
| トグル見出しラベル | 原則変えない | ON/OFF はコントロール自身で示す |
| 補助状態アイコン | 変わってよい | ただし補助状態だけを表す |

- 必須: 状態変化に伴い色が変わる対象を、状態主体の要素へ限定する。
- 禁止: ひとつの状態変化で、説明文、見出し、背景、補助情報まで一斉に同色化すること。

## 背景 / 面 / 前景の関係

- 必須: `背景`、`カード面`、`テキスト/アイコン前景` は別レイヤーとして扱う。
- 必須: 状態色は原則 `前景` にのみ使う。
- 必須: 背景やカード面は状態の意味ではなく、情報のまとまりを表す。
- 禁止: success / warning / danger をカード面全体に常用すること。
- 許可条件付き: エラーカードや破壊確認など、面自体が状態意味を持つ時のみ限定的に状態面を使える。

## macOS Active / Inactive 時の色規約

- 必須: macOS で app 非アクティブ時は、`textSecondary` と material の強度を下げてよい。
- 必須: 非アクティブ時も `danger`、`warning`、`success` の意味そのものは維持する。
- 必須: 非アクティブ時は「読めるが強すぎない」方向へ調整する。
- 禁止: 非アクティブ化で neutral に寄せて状態意味を消すこと。
- 禁止: success / danger などの状態差を消して全部同じ灰色にすること。

## アクセシビリティ仕様

- 必須: アイコン単体ボタンに accessibilityLabel を設定する。
- 必須: 状態値を持つ要素に accessibilityValue を設定する。
- 必須: accessibilityIdentifier はテスト識別専用で運用する。
- 必須: 音量、通信、ミュート、エラーは色以外でも判別可能にする。
- 必須: Dynamic Type 拡大時の文字切れを許容しない。
- 必須: タップ/クリック可能領域は iOS 基準で 44pt 以上とする。
- 必須: macOS は iOS 基準の意味を維持したうえで 28pt 以上を下限とする。

## 共通化時の禁止事項

- 禁止: 画面ごとに背景色、カード色、角丸値を個別定義すること。
- 禁止: 同種コンポーネントでトークン外の余白値を使用すること。
- 禁止: 主要導線画面へ診断情報を追加すること。
- 禁止: 将来利用を理由に未使用トークンを追加すること。
- 禁止: 標準 SwiftUI コンポーネントで意味を満たせるのに、独自のカスタム UI を導入すること。
- 禁止: Blank / Empty / Loading / Partial / Error / Disabled の意味を混同した状態表現。

## ばらつき是正の確定ルール

| ばらつき対象 | 旧状態 | 新ルール | 強制度 |
|---|---|---|---|
| 背景色 | 画面ごとに判断が分散 | 主要導線画面は callScreenBackground、診断導線画面はシステム背景、設定導線画面は grouped Form 背景に固定 | 必須 |
| カード面 | regularMaterial、thinMaterial、background が混在 | 通話系は cardMaterial、診断系は diagnosticsCardMaterial、補助パネルのみ panelSurface を許可 | 必須 |
| 項目色 | 状態色と補助色の使い分けが文脈依存 | 状態は success/warning/danger/neutral/info、補助情報は textSecondary/textTertiary に固定 | 必須 |
| 角丸 | 実質8中心だが規約化不足 | カード系はすべて radiusCard=8 に固定 | 必須 |
| 間隔 | 近い値が混在 | spacing は spaceXXS〜spaceScreen のみ使用可 | 必須 |

## 強制可能なレビュー基準

以下をすべて満たすまでPRを完了扱いにしない。

- 背景、カード面、項目色、角丸、余白がトークン定義だけで説明できる。
- 新規UIでトークン外の固定値が追加されていない。
- iOS で定義した基準値が先に存在し、macOS 側がその意味を保持している。
- 主要ボタンと補助ボタンの高さがトークン下限を下回っていない。
- 背景面とカード面の境界が視認できる。
- 主要導線画面内の空状態が同一ルールで表現されている。
- 主要導線画面と設定導線画面で caption 系のみを主要状態表示に使っていない。
- 状態表示が色単独になっていない。
- アイコン単体ボタンに accessibilityLabel がある。
- 状態要素に accessibilityValue がある。
- 主要導線画面へ診断情報が混入していない。
- 設定導線画面のセクション分離と順序が維持されている。
- 使用シンボルがシンボルファミリー意味規約に違反していない。
- 是正定義の旧シンボルが同用途で残存していない。
- variableValue 対応シンボルが固定表示で実装されていない。
- マルチカラー対応シンボルが .monochrome で描画されていない。
- SwiftUI 標準コンポーネントで成立する箇所に不要なカスタム UI が入っていない。
- Blank / Empty / Loading / Partial / Error / Disabled の状態種別が仕様上区別されている。
- macOS の foreground / background 差が意味差ではなく視覚強度差として扱われている。

| 観点 | 現行状態 | 統一方針 |
