# RideIntercom デザインシステム

本ドキュメントは、現行画面で使用している見た目・レイアウト・コントロール仕様を共通化し、実装を強制可能にするための正本である。
本ドキュメントに違反するUI変更は、実装済みであっても不合格とする。

## 強制度

- 必須: 実装時に必ず満たす。未達はマージ不可。
- 禁止: 実装してはならない。検出時は即修正。
- 許可条件付き: 明示した条件に一致する場合のみ許可。

## 適用範囲

- 対象画面: Call、Diagnostics、Settings。
- 対象要素: 背景、カード、項目色、角丸、余白、文字、ボタン、タブ、Form Section、メーター、状態表示、アクセシビリティ。
- 対象プラットフォーム: iOS、macOS。
- 基準プラットフォーム: iOS。
- macOS適用方針: iOS基準の構造、状態、アクセシビリティ、意味を維持したうえで、macOS固有の入力・ウィンドウ特性に追従する。

## iOS基準定義

- 必須: 本ドキュメント内の値、トークン、レイアウト規約、文字規約は iOS 実装を基準値とする。
- 必須: 新規UI仕様は iOS で成立する具体値を先に定義し、その後に macOS へ同等意味で展開する。
- 必須: macOS 対応は iOS 基準を崩さない範囲で行い、意味差分を生む見た目差分を追加しない。
- 禁止: macOS 側の都合を理由に iOS 基準のトークン値や状態表現を変更すること。

## 実装原則

- 必須: iOS 実装を正とし、macOS は同一の画面構造、同一の状態管理、同一のアクセシビリティIDで追従する。
- 必須: OS差分は Adapter 層で吸収し、View へOS条件分岐を入れない。
- 必須: Call 画面は通話に必要な情報と操作のみを表示し、診断情報は Diagnostics に集約する。
- 必須: Settings は Form と Section の標準構成を維持し、Audio I/O、Transmit Codec、Audio Check を分離する。
- 必須: 状態伝達は色のみで完結させない。テキスト、アイコン、accessibility label/value を併用する。

## アイコン運用

- アプリ内で使用するSF Symbolsの実体定義・シンボル規約・是正定義はすべて [アイコン定義](./アイコン定義.md) を正本とする。

### テキスト運用ルール

- 必須: タイトル系は textNavigationTitle または textSectionTitle 以外を使わない。
- 必須: ラベル系は textSubheadlineStrong、textCaptionStrong、textCaption のいずれかを用途で選ぶ。
- 必須: 補助情報は textSecondary または textTertiary を使う。
- 必須: 異常状態の文言は danger を使う。
- 必須: 診断値、メーター値、パーセンテージなど比較対象となる数値は monospacedDigit を使う。
- 必須: 長文になりうる項目は lineLimitFlexible を使い、固定1行にしない。
- 許可条件付き: 狭幅時に限り、scaleFactorTight を短文項目へ適用できる。

## コントロール共通仕様

### 主要ボタン

- 必須: 実装は appProminentButtonStyle() を使用する。
- 必須: スタイルは .borderedProminent、背景は buttonProminentBackground、文字色は buttonProminentForeground を使用する。
- 必須: サイズは .controlSize(.large) とする。

### 補助ボタン

- 必須: 補助操作は .buttonStyle(.bordered) を使用する。
- 必須: 状態反転型は danger と accentColor の2値で切り替える。

### アイコン単体ボタン

- 必須: 押下領域は最小 44x44。
- 必須: accessibilityLabel を設定する。
- 必須: 状態を持つ場合は accessibilityValue を設定する。

### スライダー

- 必須: 音量系は 0...1 の正規化範囲を使用する。
- 必須: 数値表示を併記する。
- 必須: 閾値系は最小値、最大値、スナップ間隔を固定し表示する。

### トグル

- 必須: 環境依存機能は利用可能なときのみ表示する。

### リスト行とカード

- 必須: リスト行は行全体タップ可能。
- 必須: 削除操作は swipeActions と contextMenu の両方を提供する。
- 必須: カードは padding(12) と radiusCard を使用する。

## 画面コンテナ共通仕様

### アプリ全体

- 必須: ルート構成は TabView と NavigationStack を維持する。
- 必須: タブは Call、Diagnostics、Settings の3固定。

### Call 画面

- 必須: 背景は callScreenBackground を使用する。
- 必須: 主要コンテナは ScrollView と VStack、標準縦間隔は spaceScreen。
- 必須: 通話ヘッダー、参加者カード、空状態カードは cardMaterial、cardBorder、radiusCard、borderWidthCard を使用する。

### Diagnostics 画面

- 必須: 診断行は diagnosticsCardMaterial と radiusCard でカード化する。
- 必須: 診断本文は textBodyMono を使用する。
- 必須: パイプラインは5ステップ固定、等幅配置、接続子は sizeConnector。

### Settings 画面

- 必須: Form は .grouped を使用する。
- 必須: Section 順序は Audio I/O、Transmit Codec、Audio Check。
- 必須: デバイス選択は .menu、少数択は .segmented を使用する。

## 状態表示仕様

| 状態種別 | 色 | 補助表現 | 運用ルール |
|---|---|---|---|
| 接続成功 | success | テキスト（Connected等）+ アイコン | 色単独で表現しない |
| 接続中/待機 | warning | テキスト（Connecting/Waiting等）+ アイコン | 色単独で表現しない |
| 未接続/非稼働 | neutral | テキスト（Idle/Offline等）+ アイコン | 色単独で表現しない |
| 異常/失敗/ミュート | danger | テキスト（Error/Muted等）+ アイコン | 色単独で表現しない |
| 情報提示/受信中 | info | テキスト + アイコン | 注意喚起色と混同しない |

- 必須: 状態行には色、アイコン、文言を同時に含める。
- 禁止: 状態意味を色だけで表現する実装。

## アクセシビリティ仕様

- 必須: アイコン単体ボタンに accessibilityLabel を設定する。
- 必須: 状態値を持つ要素に accessibilityValue を設定する。
- 必須: accessibilityIdentifier はテスト識別専用で運用する。
- 必須: 音量、通信、ミュート、エラーは色以外でも判別可能にする。
- 必須: Dynamic Type 拡大時の文字切れを許容しない。
- 必須: タップ/クリック可能領域は iOS 基準で 44pt 以上とする。
- 必須: macOS は iOS 基準の意味を維持したうえで 28pt 以上を下限とする。

## 共通化時の禁止事項

- 禁止: 画面ごとに背景色、カード色、角丸値を個別定義すること。
- 禁止: 同種コンポーネントでトークン外の余白値を使用すること。
- 禁止: Call 画面へ診断情報を追加すること。
- 禁止: 将来利用を理由に未使用トークンを追加すること。

## ばらつき是正の確定ルール

| ばらつき対象 | 旧状態 | 新ルール | 強制度 |
|---|---|---|---|
| 背景色 | 画面ごとに判断が分散 | Call は callScreenBackground、Diagnostics はシステム背景、Settings は grouped Form 背景に固定 | 必須 |
| カード面 | regularMaterial、thinMaterial、background が混在 | 通話系は cardMaterial、診断系は diagnosticsCardMaterial、補助パネルのみ panelSurface を許可 | 必須 |
| 項目色 | 状態色と補助色の使い分けが文脈依存 | 状態は success/warning/danger/neutral/info、補助情報は textSecondary/textTertiary に固定 | 必須 |
| 角丸 | 実質8中心だが規約化不足 | カード系はすべて radiusCard=8 に固定 | 必須 |
| 間隔 | 近い値が混在 | spacing は spaceXXS〜spaceScreen のみ使用可 | 必須 |

## 強制可能なレビュー基準

以下をすべて満たすまでPRを完了扱いにしない。

- 背景、カード面、項目色、角丸、余白がトークン定義だけで説明できる。
- 新規UIでトークン外の固定値が追加されていない。
- iOS で定義した基準値が先に存在し、macOS 側がその意味を保持している。
- 状態表示が色単独になっていない。
- アイコン単体ボタンに accessibilityLabel がある。
- 状態要素に accessibilityValue がある。
- Call 画面へ診断情報が混入していない。
- Settings のセクション分離と順序が維持されている。
- 使用シンボルがシンボルファミリー意味規約に違反していない。
- 是正定義の旧シンボルが同用途で残存していない。
- variableValue 対応シンボルが固定表示で実装されていない。
- マルチカラー対応シンボルが .monochrome で描画されていない。

## 逸脱時の扱い

- 原則として例外を認めない。
- どうしても例外が必要な場合は、同時変更で次を必須とする。
- 例外理由。
- 影響範囲。
- 代替不可の根拠。
- 解除条件。
- 期限。

## 画面項目仕様との関係

- 画面ごとの項目と発現条件は [画面項目定義](./画面項目定義.md) を正本とする。
- 本ドキュメントは見た目と操作規約の正本であり、項目の有無や条件は画面項目定義に従う。

## 現行差分と統一方針

| 観点 | 現行状態 | 統一方針 |
|---|---|---|
| 背景面 | Call は `callScreenBackground`、Diagnostics は素の背景、Settings は Form 背景 | 画面種別ごとに背景ルールを固定し、同種画面での例外を禁止する |
| カード面 | `regularMaterial`、`thinMaterial`、`.background` が混在 | 通話系カード、診断系カード、補助パネルで面種別を明確に分離する |
| 角丸 | 実質 8 が中心だが定数化されていない | `radiusCard = 8` を共通トークンとして固定する |
| 間隔 | 2/3/4/6/8/10/12/14/16 が散在 | 余白トークンとして明示し、トークン外の値追加を制限する |
| ボタン | 主要ボタンと補助ボタンの使い分けはあるが規約化不足 | 主要・補助・アイコン単体の3分類を固定する |
